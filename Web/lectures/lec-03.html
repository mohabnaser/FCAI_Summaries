Why OOP? — ليه بنستخدم الـ OOP؟

تنظيم الداتا بشكل يعكس الواقع

🤔 إيه المشكلة اللي الـ OOP بيحلها؟

البرامج الصغيرة ممكن نكتبها بشوية متغيرات ودوال (Variables & Functions) متبعترة. بس لما البرنامج يكبر ويبقى سيستم كامل لجامعة مثلاً، هيبقى عندنا آلاف الطلاب والكورسات. الـ OOP بيخلينا نبرمج الحاجات دي كأنها "كيانات" حقيقية (Entities) زي Student و Course. وده بيسهل قراءة وتعديل الكود.

ملحوظة مهمة للـ Backend: كل ما هنيجي نعمل Model في الـ Django بعدين، إحنا في الحقيقة بنعمل Class. المحاضرة دي هي الأساس الحقيقي للـ Django!
🏛️ Class vs Object — الفرق بين التصميم والنسخة

الـ Class (التصميم / Blueprint)

هو الرسمة أو "الأسطمبة". مفيش فيها داتا حقيقية، بس بتعرّف إيه البيانات اللي المفروض تكون موجودة وايه الأفعال الممكنة. (زي فكرة "نموذج فاضي لبيانات الطالب").

الـ Object (النسخة / Instance)

هو الحاجة الحقيقية اللي اتعملت من الأسطمبة دي. يعني "الطالب علي" أو "الطالبة منى"، كل واحد فيهم Object شايل بيانات مختلفة بس ليهم نفس الهيكل.

Attributes & Methods — البيانات والأفعال

الكلاس بيتكون من إيه من جوه؟

🧩 إيه هما الـ Attributes والـ Methods؟
  • Attributes: هي "البيانات" اللي جوه الـ Object (يعني الكائن ده بيعرف إيه؟ زي اسمه وعمره).
  • Methods: هي "الدوال" المكتوبة جوه الـ Class (يعني الكائن ده بيعرف يعمل إيه؟ زي إنه يطبع بياناته).
✨ إنشاء كلاس بسيط و Objects منه

عشان نكتب كلاس في بايثون بنستخدم كلمة class، وغالباً اسم الكلاس بيبدأ بحرف كابيتال (PascalCase).

# 1. إنشاء كلاس فاضي
class Student:
    pass

# 2. إنشاء Objects (Instantiation)
student1 = Student()
student2 = Student()

# 3. إضافة Attributes يدوياً (دي طريقة مش مفضلة بس ممكنة)
student1.name = "Mona"
student1.level = 2

print(student1.name)  # هيطبع Mona
ليه الطريقة اليدوية وحشة؟ لأنك ممكن تنسى تضيف level لـ student2 وساعتها الكود هيضرب Error. عشان كده بنستخدم الـ Constructor!

Constructor & Self — دالة البناء

ازاي ننظم ولادة الأوبجكت؟

🏗️ دالة البناء __init__

دي دالة بتشتغل أوتوماتيك أول ما تعمل Object جديد، وظيفتها إنها تضمن إن الـ Object ده اتخلق وهو معاه كل الداتا الأساسية بتاعته.

class Student:
    # הConstructor
    def __init__(self, name, level, department="General"):
        self.name = name
        self.level = level
        self.department = department

# بمجرد ما بنعمل كده، بايثون بتنادي __init__ لوحدها
student1 = Student("Mona", 2, "IS")
student2 = Student("Ali", 2) # هياخد الـ default ("General")
👤 يعني إيه كلمة self ؟

كلمة self بترمز للـ Object الحالي اللي بيتعمله إنشاء أو اللي بينادي الدالة. لما نقول self.name = name كأننا بنقول "يا بايثون، خد القيمة اللي جيالك وحطها في الـ Object اللي في إيدينا دلوقتي".

خطأ شائع جداً: لو كتبت دالة جوه الـ Class ونسيت تحط self كأول بارامتر، بايثون هيرمي Error أول ما تيجي تنادي الدالة دي!

Inheritance & Overriding — الوراثة والتعديل

إعادة استخدام الكود بذكاء

👨‍👦 مفهوم الوراثة (Inheritance)

لو عندنا كلاسين شبه بعض (زي الطالب والمدرس، الاتنين أشخاص ليهم اسم)، بدل ما نكرر الكود، بنعمل كلاس كبير (Parent) اسمه Person، ونخلي الكلاس الصغير (Child) يورث منه. بايثون بتستخدم الأقواس للوراثة class Student(Person): ومش بتستخدم كلمة inherits!

class Person:
    def __init__(self, name):
        self.name = name
    def introduce(self):
        print("I am a person")

class Student(Person):
    def __init__(self, name, level):
        # super() بتنادي دالة الأب عشان نبني الجزء المشترك (الاسم)
        super().__init__(name)
        self.level = level
    
    # Method Overriding: بنعيد كتابة الدالة عشان نغير سلوكها للطالب بس
    def introduce(self):
        print("I am a student")
🔍 إيه لازمة دالة super() ؟

دالة super() بتديلك صلاحية توصل لدوال الكلاس الأب (Parent) وتشغلها جوه الكلاس الابن (Child). غالباً بنستخدمها عشان نشغل الـ __init__ بتاعة الأب ومانكتبش كود الداتا الأساسية مرتين.

🔒 Private vs Public في بايثون

بايثون مافيهاش حاجة اسمها Private بالمعنى الحرفي زي Java! بس متعارف عليه بين المبرمجين إن لو اسم الـ Attribute بيبدأ بـ Underscore _name، يبقى ده للاستخدام الداخلي ومحدش من بره المفروض يغيره (Convention مش قانون).

File Handling — التعامل مع الملفات

حفظ الداتا بره الرامات

📂 ليه بنستخدم الملفات في الـ Backend؟

المتغيرات بتتمسح أول ما البرنامج يقفل. لو عايز تحفظ Logs (سجلات) أو Settings أو بيانات يوزرز بشكل مبدئي قبل الداتابيز، بنستخدم الملفات. بنفتح الملف بدالة open(filename, mode).

Modes (أوضاع فتح الملف)

  • "r" (Read): للقراءة بس.
  • "w" (Write): للكتابة. (بيمسح أي كلام قديم ويكتب مكانه!)
  • "a" (Append): بيضيف الكلام الجديد في آخر الملف بدون مسح.

Reading Methods

  • read(): بيقرأ الملف كله كـ String واحد.
  • readline(): بيقرأ سطر واحد بس.
  • readlines(): بيقرأ السطور ويرجعها في List.
🛡️ الأسلوب الأأمن: with open(...)

لو استخدمت open() العادية، لازم تقفل الملف في الآخر بـ file.close(). الأفضل نستخدم with open() لأنها بتقفل الملف أوتوماتيك أول ما البلوك يخلص، حتى لو حصل Error في النص.

# كتابة الملف باستخدام with (طريقة آمنة)
with open("students.txt", "w") as file:
    file.write("Mona\n")  # الـ \n بتعمل سطر جديد
    file.write("Ali\n")
# هنا الملف اتقفل لوحده خلاص!

Errors & JSON — أخطاء الملفات والجيسون

ازاي بنبعت ونستقبل داتا في الويب؟

🐞 أشهر أخطاء الملفات
  • FileNotFoundError: بتظهر لو استخدمت وضع "r" لملف مش موجود أصلاً، أو كاتب مساره غلط. عشان نتجنب الكراش، بنحطها جوه try/except.
  • PermissionError: لو البرنامج مش واخد صلاحية يدخل على الملف ده أو يكتب فيه.
📦 الـ JSON وأهميته في الـ Backend

الـ JSON (JavaScript Object Notation) هي طريقة بنرتب بيها البيانات عشان ننقلها بسهولة بين الـ Backend والـ Frontend. شكلها في بايثون شبه الـ Dictionary بالظبط.

لما يكون عندك Object، مينفعش ترميه زي ما هو في ملف JSON! لازم الأول تحوله لـ Dictionary عن طريق دالة بتعملها زي to_dict() (العملية دي اسمها Serialization).

import json

student = {"name": "Mona", "level": 2}

# json.dump(): بتحفظ الـ dict في ملف JSON
with open("data.json", "w") as f:
    json.dump(student, f, indent=4)  # indent بتخلي شكله منسق ومقروء

# json.load(): بتقرأ من ملف JSON وترجعه لـ dict في بايثون
with open("data.json", "r") as f:
    data = json.load(f)
    print(data["name"])

Q1 Final 2021
Which statement is correct about this Python code?
class Name:
  def __init__(self, name):
    self.name = name
name1 = Name('ABC')
name2 = name1
A
It will throw an error as multiple references to the same object is not possible.
B
id(name1) and id(name2) will have the same value.
C
They are completely separate objects in memory.
D
All of the above
✅ الإجابة الصح: B — id(name1) and id(name2) will have the same value
لما بتعمل name2 = name1 إنت مش بتعمل نسخة جديدة من الـ Object! إنت بس بتعمل "إشارة" جديدة (Reference) لنفس المكان في الذاكرة. فالاتنين بيكون ليهم نفس الـ ID بالظبط.
