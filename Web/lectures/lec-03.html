Classes & Objects

The building blocks of OOP in Python

📦 Defining a Class
class Student:
    def __init__(self, name, gpa):  # constructor — runs when object is created
        self.name = name            # instance attribute
        self.gpa = gpa

    def greet(self):
        print(f"Hello, I am {self.name}")

s1 = Student("Ali", 3.8)
s2 = s1          # s2 is a REFERENCE to the same object!
# id(s1) == id(s2) → True
Exam 2021: When you do name2 = name1, both variables reference the same object. id(name1) == id(name2) = True.
Key: self is the first parameter of every method — it refers to the current object instance.

Inheritance & Polymorphism

Reusing and overriding code

👪 Inheritance Example
class Animal:
    def speak(self):
        return "..."

class Dog(Animal):        # Dog inherits from Animal
    def speak(self):      # Override (polymorphism!)
        return "Woof!"

class Cat(Animal):
    def speak(self):
        return "Meow!"

d = Dog()
print(d.speak())  # "Woof!" — uses Dog's version
Polymorphism: Same method name (speak), different behavior depending on the class. Very powerful for building flexible systems.

The 4 Pillars of OOP

Encapsulation · Inheritance · Polymorphism · Abstraction

🏛️ Four OOP Pillars

🔒 Encapsulation

Bundle data + methods together. Hide internal state from outside. Use _private or __private to restrict access.

🔗 Inheritance

Child class inherits attributes and methods from parent class. Promotes code reuse — don't repeat yourself!

🎭 Polymorphism

Same method name, different behavior in different classes. Override parent methods to customize behavior.

🎭 Abstraction

Hide complex implementation details. Show only the essential interface to the user.

File Handling

Read, write, and append files in Python

📄 File Operations
# Read a file
with open("data.txt", "r") as f:
    content = f.read()

# Write to a file (overwrites!)
with open("output.txt", "w") as f:
    f.write("Hello World")

# Append to a file (does NOT overwrite)
with open("log.txt", "a") as f:
    f.write("new line\n")
Best Practice: Always use with — it automatically closes the file even if an error occurs. No need to call f.close().
ModeMeaningCreates if not exists?
"r"Read only❌ No (error)
"w"Write (overwrites)✅ Yes
"a"Append (adds to end)✅ Yes
"rb"Read binary❌ No

Q1 Final 2021
Consider: name1 = Name('ABC'); name2 = name1. Which statement is correct?
A
Error — multiple references to same object not possible
B
id(name1) and id(name2) will have the same value
C
name2 is a copy of name1
D
Error — can't assign object to variable
✅ Correct: B — id(name1) == id(name2)
name2 = name1 creates a reference, NOT a copy. Both variables point to the exact same object in memory. Their id() values are identical. لما بتعمل name2 = name1 مش بتعمل نسخة — بتعمل اسم تاني لنفس الـ object. الاتنين بيشيروا لنفس المكان في الـ memory. ده زي ما لو قلت "محمد" و "كيميائي" — اتنين أسماء لنفس الشخص.

Q3 Final 2024
Which OOP pillar means "same method name, different behavior in different classes"?
A
Polymorphism
B
Encapsulation
C
Abstraction
D
Inheritance
✅ Correct: A — Polymorphism
Polymorphism = one interface, many forms. The same method name behaves differently across classes. Like speak() in Dog vs Cat. Polymorphism = نفس الاسم، تصرف مختلف. زي طريقة speak() في الـ Dog بترجع "Woof" وفي الـ Cat بترجع "Meow" — نفس الاسم بس تصرف مختلف.
Q4 Final 2024
Which file mode in Python ADDS content without overwriting existing content?
A
"w" (write)
B
"r" (read)
C
"a" (append)
D
"rw" (read-write)
✅ Correct: C — "a" (append)
Mode "a" opens the file and adds content at the END without deleting existing content. Mode "w" overwrites everything. "a" (append) = بيضيف في الآخر من غير ما يمسح اللي موجود. "w" (write) = بيمسح كل حاجة ويكتب من الأول. اتأكد تعرف الفرق ده!
