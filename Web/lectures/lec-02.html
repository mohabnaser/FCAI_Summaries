Operators & Conditions — العمليات والشروط

المنطق واتخاذ القرارات

➕ العمليات الحسابية (Arithmetic Operators)

الجمع +، الطرح -، الضرب *، القسمة /، باقي القسمة %، والأسس **.

خد بالك في الامتحان!
القسمة العادية / دايماً بتطلع float. يعني 10 / 2 = 5.0 مش 5. لو عاوز الناتج رقم صحيح، استخدم القسمة الصحيحة // (زي 10 // 3 = 3).
🔍 المقارنة والمنطق (Comparison & Logical)
  • المقارنة: اليساوي == (للمقارنة، عكس = اللي بتدي قيمة)، != (لا يساوي)، >، <، >=، <=. نتيجتهم دايماً True أو False.
  • المنطق: and (لازم الشرطين يبقوا صح)، or (واحد بس يكفي)، not (بتعكس النتيجة).
🔀 الجمل الشرطية (if, elif, else) والمسافات (Indentation)

بايثون بتعتمد بشكل أساسي على الـ Indentation (المسافات البادئة). مفيش أقواس {} زي باقي اللغات. لو نسيت المسافة الكود هيضرب IndentationError.

mark = 85
if mark >= 90:
    print("A")      # لازم مسافة هنا (غالباً 4 مسافات)
elif mark >= 80:    # elif دي زي else if في اللغات التانية
    print("B")
else:
    print("C")

Loops — التكرار

تنفيذ الكود أكتر من مرة بشكل أوتوماتيكي

🔁 for loop & range()

الـ for بتستخدم عشان تلف على مجموعة عناصر أو تكرر حاجة عدد معين من المرات. وبنستخدم معاها دالة range().

# بتطبع من 0 لـ 4 (الرقم الأخير excluded)
for i in range(5):
    print(i)

# بتطبع من 1 لـ 5
for i in range(1, 6):
    print(i)
فخ امتحان 2024! لو حطيت فاصلة منقوطة ; بعد الـ for (مثلاً for i in range(5);) ده هيطلع SyntaxError. بايثون بتستخدم النقطتين :.
🛑 break و continue
  • break: بتكسر اللوب وتخرج براه خالص.
  • continue: بتعمل skip للفة الحالية وتكمل اللي بعدها.
  • pass: دي Placeholder مبنعملش حاجة، بنحطها بس عشان الكود ميضربش لو مفيش كود جوه الـ block.

Data Structures / Collections — هياكل البيانات

إزاي نخزن بيانات كتير في متغير واحد؟

📋 مقارنة بين القوائم والمجموعات (الخلاصة للامتحان)

List [ ]

  • مرتبة (Ordered) ولها Index.
  • ينفع تعدل فيها (Mutable).
  • تقبل التكرار (Duplicates).
  • بتضيف بـ append()، وتمسح الآخر بـ pop()، وتدمج بـ extend().

Tuple ( )

  • مرتبة (Ordered) ولها Index.
  • مستحيل تتعدل (Immutable).
  • مثال: x = ("apple", "banana"). لو عنصر واحد لازم نحط فاصلة y = ("apple",) عشان تتعرف كـ Tuple!

Set { }

  • عشوائية وغير مرتبة (Unordered & Unindexed).
  • لا تقبل التكرار أبداً.
  • لو عملت pop() هتمسح عنصر عشوائي!
  • عشان تضيف عناصر من set تانية بنستخدم update() مش extend!

Dictionary { }

  • أزواج من Key: Value.
  • الـ Keys لازم تكون فريدة (Unique).
  • لو ضفت Key موجود أصلاً، القيمة الجديدة هتمسح القديمة (بدون Error).
✂️ List Slicing & Operations

الـ Slicing في بايثون بيتبع قاعدة: list[start:end] بحيث بيبدأ من الـ start (included) ويقف قبل الـ end (excluded).

lst = [0, 1, 2, 3, 4]
print(lst[1:3])   # هيطبع [1, 2] لأن الـ index 3 مش معانا!

txt = "Hello, My Friend"
print(txt[:8])    # هيطبع "Hello, M" (بيبدأ من الأول لحد index 7)
🔄 الترتيب (Sorting) وفخ الحروف

دالة sort() بترتب العناصر. بس بالنسبة للحروف، الحروف الـ Capital بتيجي قبل الـ Small (حسب كود الـ ASCII).

x = ['C', 'b', 'a']
x.sort()
print(x)  # هيطبع ['C', 'a', 'b'] لأن الـ C بتيجي الأول!

# عشان تتجاهل حالة الأحرف بنستخدم key=str.lower
y = ['C', 'b', 'A']
y.sort(key=str.lower)
print(y)  # هيطبع ['A', 'b', 'C'] بيحافظ على شكلهم الأصلي بعد الترتيب الأبجدي.
📦 Tuple Unpacking

ممكن نفكك الـ Tuple ونحط القيم في متغيرات. ولو عدد المتغيرات مش بيساوي القيم بيضرب Error، إلا لو استخدمنا النجمة * اللي بتجمع الباقي في List!

tup = (1, 2, 3, 4)
(*k,) = tup
print(k)  # النجمة جمعت كل العناصر في list، فهتطبع [1, 2, 3, 4]

# بس لو المتغيرات الإجبارية أكتر من العناصر؟ هيضرب ValueError!
newtuple = (1, 2, 3)
(x, *k, y, z, m) = newtuple  # Error! مفيش عناصر تكفي x, y, z, m

Functions & Scope — الدوال والنطاق

تنظيم الكود وإعادة استخدامه

⚙️ تعريف الدالة وكلمة def

الدالة (Function) بلوك كود بينفذ مهمة معينة ومش بيشتغل إلا لو عملتله Call. بنعرفه بكلمة def. وبنستخدم return عشان نرجع قيمة (دي غير print اللي بتعرض بس).

✨ *args و **kwargs
  • *args: بتجمع أي عدد من البارامترات في Tuple.
  • **kwargs: بتجمع البارامترات اللي متسمية (Keyword arguments) في Dictionary.
def info(**person):
    print(person["name"])  # بتتعامل معاها كـ dict عادي

info(name="Ahmed", age=10) # هيطبع Ahmed بس!
λ Lambda Functions (الدوال المجهولة)

الـ Lambda هي دالة سريعة بتكتب في سطر واحد بدون اسم. فكرتها lambda arguments: expression.

y = lambda x: x / 20
print(y(100))  # هيطبع 5.0 (لأن القسمة دايماً float!)

Modules, Errors & Files — الملفات والأخطاء

التعامل مع بيئة العمل الخارجية

🧩 الـ Modules والاستيراد

الـ Module هو ملف بايثون فيه كود جاهز (زي مكتبة math). بنعمله import عشان نوفر على نفسنا كتابة الكود من الصفر.

🐛 التعامل مع الأخطاء (try / except)

عشان الكود ميقفلش فجأة لو حصل Error بنستخدم try و except. أنواع الأخطاء المشهورة: SyntaxError (غلطة في الكتابة)، NameError (متغير مش موجود)، IndentationError (مشكلة في المسافات).

📄 التعامل مع الملفات (File Handling)

بنستخدم دالة open(filename, mode):

  • "r" (Read): للقراءة فقط.
  • "w" (Write): للكتابة، وبتمسح أي كلام قديم في الملف! لو الملف مش موجود بتعمله إنشاء.
