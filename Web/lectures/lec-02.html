Python is dynamically typed — no need to declare types!

TypeExampleKey Note
intx = 10Integer number
floatx = 3.14Division / ALWAYS returns float: 10/2 = 5.0
strx = "Ahmed"Text — immutable

Collections

List · Tuple · Set · Dictionary

📋 The Four Collection Types

List []

  • Ordered ✅
  • Mutable ✅ (can change)
  • Allows duplicates ✅
  • Indexed by position [0]
lst = [1, 2, 3]
lst.append(4)   # → [1, 2, 3, 4]
lst.pop()       # removes last

Tuple ()

  • Ordered ✅
  • Immutable ❌ (can't change)
  • Allows duplicates ✅
  • Faster than list
t = (1, 2, 3)
# t[0] = 10 → TypeError!

Set {}

  • Unordered ❌
  • NOT indexed ❌
  • NO duplicates ❌
  • pop() = random element!
s = {1, 2, 3}
s.pop()  # removes RANDOM!

Dictionary {}

  • Key-value pairs
  • Unique keys only
  • Duplicate key → last wins
  • Mutable ✅
d = {"name": "Ali", "age": 20}
d["name"]  # → "Ali"
Exam Trap (2021 & 2024): A set is unordered and NOT indexed. Sets also do NOT allow duplicates — adding a duplicate just ignores it. pop() on a set removes a random element!
📖 Dictionary — Duplicate Keys & References
# Duplicate keys — LAST VALUE WINS (no error!)
x = {"name": "Ahmed", "id": "1", "mobile": 123456789, "mobile": 987654321}
print(x)
# → {'name': 'Ahmed', 'id': '1', 'mobile': 987654321}

# REFERENCE vs COPY — 2024 exam question!
d = {"name": "Mona"}
y = d           # y is a REFERENCE to same dict
d["name"] = "Sara"
print(y)        # → {'name': 'Sara'}  ← y changed too!

y2 = d.copy()   # y2 is an INDEPENDENT copy (lowercase copy!)
d["name"] = "Ali"
print(y2)       # → {'name': 'Sara'}  ← y2 did NOT change
2024 Exam: y = d makes y a reference — any change to d ALSO changes y. Use d.copy() (lowercase!) for independent copy. Copy() with capital C is a TypeError — Python is case-sensitive!
✂️ List Slicing
lst = [0, 1, 2, 3, 4, 5]
lst[1:3]    # → [1, 2]    (includes index 1, EXCLUDES index 3)
lst[:3]     # → [0, 1, 2] (from start to index 2)
lst[2:]     # → [2, 3, 4, 5] (from index 2 to end)
lst[-1]     # → 5          (last element)
Rule: Slicing includes the first index, excludes the last index. If no bound → assumes start/end of list.
🔢 List Sort — Nested Lists
lst = []
nums = [15, 6]
lst.append(nums)           # lst = [[15, 6]]
nums = [10, 30, 20]
lst.append(nums)           # lst = [[15, 6], [10, 30, 20]]
lst.sort()
print(lst)
# → [[10, 30, 20], [15, 6]]
# sort() compares FIRST elements only: 10 < 15
# Inner lists are NOT sorted

Control Flow

if/elif/else, loops — and the indentation trap

⚠️ Indentation is MANDATORY in Python

✅ Correct — Only 2nd print runs:

if 2 < 1:
    print("Two is less than one!")
print("Always prints")

Output: Always prints

❌ Wrong indentation = Error:

if 2 < 1:
print("Will crash!")
# IndentationError!
2024 Exam: "Notice the incorrect indentation between the 2 print functions." — Be very careful about which print is inside or outside the if block.
🔄 For Loop with Semicolon Trap
# DANGER: semicolon after for loop header!
for i in range(3);    # SyntaxError — semicolons NOT used in Python for loops

# But in some tricky questions, an empty loop body
for i in range(3):
    pass
print("done")
2024 Exam: "Note the ; after the for loop" — a semicolon after a for statement is a SyntaxError in Python. Python uses colons (:) not semicolons (;) for block headers.
🔢 Python Arithmetic Types
y = 2
z = 3
print(y + z)    # → 5  (both ints → int result)

print(10 / 2)   # → 5.0 (division ALWAYS float!)
print(10 // 2)  # → 5   (integer division, use //)
2024 Exam: "y and z are ints and are added normally — The answer is 5." Adding two ints gives int. But division / always gives float even with two ints.

Functions

def, *args, **kwargs, scope — و كمان Lambda!

⚙️ *args and **kwargs
# *args → packs positional arguments into a TUPLE
def greet(*names):
    for name in names:
        print(f"Hello {name}")

greet("Ali", "Mona", "Ahmed")
# → Hello Ali / Hello Mona / Hello Ahmed

# **kwargs → packs keyword arguments into a DICT
def f(**person):
    print("Person's name is " + person["name"])

f(name="Ahmed", age=10)
# → Person's name is Ahmed
Exam 2021: **kwargs sends arguments as a dictionary. The function accesses them with person["name"] like a regular dict.
λ

Lambda & Advanced

Anonymous functions, map, filter — مهمين جداً!

λ Lambda Functions
y = lambda x: x / 20
print(y(100))
# → 5.0  (NOT 5! Division always float in Python)

double = lambda n: n * 2
print(double(4))
# → 8 (multiplication of ints → int)
Classic Trap: Lambda with division gives float. Lambda with multiplication of ints gives int.
