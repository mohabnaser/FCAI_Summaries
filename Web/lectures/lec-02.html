Why Python? — ليه Python للـ Backend؟

إعداد البيئة خطوة خطوة

💡 ليه Python تحديداً؟

Readable (مقروءة): الكود بيتكلم بلغة قريبة من اللغة الإنجليزية العادية — تقدر تركز على المنطق بدل الأقواس والفواصل.

Productive (إنتاجية عالية): تقدر تبني Prototype بسرعة وتختبر الأفكار في وقت أقل بكتير من لغات تانية.

Ecosystem ضخم: عندها Django و Flask و FastAPI وآلاف المكتبات الجاهزة.

مناسبة للتعليم: بتخلي الطالب يتعلم المنطق البرمجي بسهولة قبل ما يتعامل مع تعقيدات لغات تانية.

🔗 ربط بالمستقبل: لما تشتغل مع Django، هتلاقي إن كل الـ Views عبارة عن Functions بايثون. كل الـ Models عبارة عن Classes بايثون. كل حاجة هتتعلمها النهارده هتشوفها فيه.

Variables & Data Types — المتغيرات وأنواع البيانات

أساس أي برنامج — إزاي بنحفظ المعلومات؟

📦 Variables — المتغيرات

المتغير (Variable) زي الصندوق المتسمّي — بنحط فيه قيمة ونستخدمها بعدين. علامة = هي الـ Assignment Operator، يعني "حط القيمة دي في المتغير ده".

PYTHON — Variables
student_name = "Mona"     # str — نص
age = 20                 # int — رقم صحيح
gpa = 3.8                # float — رقم عشري
is_active = True         # bool — صح أو غلط
course = "Backend Python"

print(student_name)
print(type(age))         # بيطبع: <class 'int'>
📌 دالة type(): بتعرفك نوع المتغير — استخدمها لما الكود بيطلع نتيجة غريبة. كتير من الأخطاء بتيجي من إنك مش عارف النوع الحقيقي للقيمة.
📏 قواعد تسمية المتغيرات — Naming Rules

✅ صح

VALID NAMES
student_name = "Ali"
score1 = 90
_private = "hidden"
myVar = 5

❌ غلط

INVALID NAMES
1score = 90  # يبدأ برقم ❌
my-var = 5   # فيه شرطة ❌
my var = "x" # فيه مسافة ❌
for = 1     # كلمة محجوزة ❌
🚨 من امتحان 2025: الاسم my-var هو اسم غلط في Python — الشرطة العادية - مش مسموح بيها في أسماء المتغيرات. استخدم الشرطة السفلية my_var بدلها ✅

💡 f-Strings: الطريقة الأفضل لعرض متغيرات جوا نص هي الـ f-string: f"اسمي {name} وسني {age}" — أوضح وأنضف من الـ Concatenation بـ +.

Input / Output & Comments — الإدخال والإخراج والتعليقات

إزاي البرنامج يتكلم مع المستخدم

⌨️ input() — استقبال بيانات

دالة input() بتوقف البرنامج وتستنى المستخدم يكتب حاجة. النقطة المهمة: input() دايماً بترجع String، حتى لو المستخدم كتب رقم!

PYTHON
name = input("اكتب اسمك: ")
print(f"أهلاً {name}!")
🔄 Type Casting — تحويل النوع

عشان تحول الـ String الجاي من input() لرقم، لازم تعمل Type Casting. استخدم int() للأعداد الصحيحة وfloat() للعشرية.

# int Casting
age = int(input("السن: "))
# float Casting

print(age + 1)  # شغال ✅

Operators & Conditions — العمليات والشروط

إزاي البرنامج يفكر ويتخذ قرارات؟

➕ Arithmetic Operators — العمليات الحسابية
الرمزالاسممثالالنتيجة
+Addition10 + 313
-Subtraction10 - 37
*Multiplication10 * 330
/Division10 / 33.333... (float!)
//Integer Division10 // 33 (بدون كسر)
%Modulus (باقي القسمة)10 % 31
**Exponent (الأسس)2 ** 38
🚨 فخ الامتحان الأول: القسمة العادية / دايماً تطلع float حتى لو الناتج عدد صحيح. 10 / 2 = 5.0 وليس 5!
🔍 Comparison & Logical Operators

Comparison Operators — نتيجتهم دايماً True أو False:

PYTHON
score = 75
print(score == 75)  # True
print(score != 100) # True
print(score > 50)   # True
print(score <= 70)  # False

Logical Operators — لدمج أكتر من شرط:

PYTHON
age = 20
has_id = True

# and: لازم الاتنين صح
print(age >= 18 and has_id)  # True
# or: واحد بس يكفي
print(age < 18 or has_id)   # True
# not: يعكس النتيجة
print(not has_id)             # False
⚠️ = vs ==: علامة = بتحط قيمة في متغير (Assignment). علامة == بتقارن قيمتين (Comparison). الخلط بينهم من أشهر الأخطاء!
🔀 if / elif / else — الجمل الشرطية

Python بتعتمد على الـ Indentation (المسافات البادئة) عشان تعرف الـ Blocks. مفيش أقواس {} زي باقي اللغات. لو نسيت المسافة هيطلع IndentationError!

PYTHON — if / elif / else
mark = 87

if mark >= 90:
    print("Excellent")      # مسافة 4 بادئة ✅
elif mark >= 80:           # elif = "else if"
    print("Very Good")
elif mark >= 65:
    print("Good")
else:
    print("Need Improvement")
📌 Nested Conditions (شروط متداخلة): تقدر تحط if جوا if تاني — مفيد لحالات زي Login ثم فحص الـ Role. لو logged_in صح، فافحص هل هو admin ولا لأ.
PYTHON — Nested Conditions (مثال Backend واقعي)
logged_in = True
is_admin = False

if logged_in:
    if is_admin:
        print("افتح لوحة الأدمن")
    else:
        print("افتح لوحة الطالب")
else:
    print("من فضلك سجّل دخولك")
🚨 من امتحان 2021 — سؤال الـ Indentation:
if 2 < 1: [indented] print("Two is less than one!") [not-indented] print("Two is less than one!")
النتيجة: الـ print الأولى مش هتتنفذ (الشرط غلط)، الثانية هتتنفذ دايماً لأنها بره الـ if.

Loops — for & while

تنفيذ نفس الكود أكتر من مرة بشكل أوتوماتيكي

🔁 for Loop

الـ for بنستخدمه لما عارف عدد التكرارات. دالة range() بتولد أرقام:

PYTHON
# range(5) = 0, 1, 2, 3, 4
for i in range(5):
    print(i)

# range(1,6) = 1, 2, 3, 4, 5
for i in range(1, 6):
    print(i)

# Loop على عناصر قائمة
students = ["Ali", "Mona", "Sara"]
for student in students:
    print(student)
⚡ من امتحان 2024: لو كتبت for i in range(5); بفاصلة منقوطة بدل النقطتين فوق بعض، هيطلع SyntaxError!
🔄 while Loop

الـ while بيكمل طالما الشرط True. مهم دايماً تعدّل المتغير جوا اللوب عشان متدخلش في Loop لا نهائي!

PYTHON
count = 1

while count <= 5:
    print(count)
    count += 1   # مهم! عشان اللوب يوقف

# Output: 1, 2, 3, 4, 5
🛑 break, continue, pass

🔴 break

  • يخرج من اللوب فوراً
  • يوقف كل التكرار

🟡 continue

  • يتخطى الفردة الحالية
  • يكمل اللوب للفردة اللي بعدها

⚪ pass

  • ما بيعملش حاجة
  • Placeholder فاضي للكود
PYTHON — break & continue
for i in range(1, 6):
    if i == 3:
        continue    # يتخطى 3
    if i == 5:
        break       # يوقف عند 5
    print(i)

# Output: 1, 2, 4

List & Tuple — القوائم والـ Tuples

هياكل بيانات مرتبة — الفرق بين المتغير والثابت

📋 List — القائمة [ ]

القائمة بتخزن عناصر بالترتيب وبتقدر تعدل فيها. كل عنصر بيبدأ من Index رقم 0.

PYTHON — List Operations
students = ["Ali", "Mona", "Sara"]
print(students[0])   # Ali
print(students[-1])  # Sara (الأخير)

students.append("Nour")  # يضيف للآخر
students.remove("Ali")  # يحذف بالقيمة
students.pop()          # يحذف ويرجع الأخير
students.pop(0)        # يحذف الأول
print(len(students))  # عدد العناصر
PYTHON — List Slicing
lst = [0, 1, 2, 3, 4]
# list[start:end] — end مش متضمنة!
print(lst[1:3])   # [1, 2]

txt = "Hello, My Friend"
print(txt[:8])   # "Hello, M"
# يبدأ من الأول حتى index 7
PYTHON — Sorting
x = ['C', 'b', 'a']
x.sort()
print(x)  # ['C', 'a', 'b'] ← C أولاً!
# الحروف Capital بتيجي قبل Small

y = ['C', 'b', 'A']
y.sort(key=str.lower)
print(y)  # ['A', 'b', 'C']
# key=str.lower: يرتب بدون اعتبار حالة الحرف
# لكن يحافظ على شكله الأصلي!
🚨 من امتحانات 2021 و2025:
• sort() على list من lists بيقارن أول عنصر فقط
• sort(key=str.lower) يرتب بدون اعتبار الـ Case لكن يحافظ على الشكل الأصلي
• pop() بدون argument تشيل آخر عنصر دايماً
📌 Tuple — ( )

الـ Tuple زي الـ List بالظبط، الفرق الوحيد إنها Immutable — مش ممكن تعدل فيها بعد ما تنشئها. بتستخدمها لما البيانات المفروض تفضل ثابتة.

PYTHON — Tuple
point = (10, 20)
print(point[0])   # 10

# Tuple بعنصر واحد — لازم الفاصلة!
t1 = ("apple",)    # ✅ Tuple
t2 = ("apple")     # ❌ String مش Tuple!

# Tuple Unpacking
tup = (1, 2, 3, 4)
(*k,) = tup
print(k)  # [1, 2, 3, 4]
🚨 من امتحان 2025 (Q70 & Q71):
(*k,) = (1,2,3,4) → يحول كل العناصر لـ List: [1, 2, 3, 4] ✅
(x, *k, y, z, m) = (1,2,3) → ValueError! المتغيرات الإجبارية (x, y, z, m) أكتر من العناصر.
⚡ من امتحان 2025: x = ("apple", "banana", "cherry") — هذه ليست List! دي Tuple لأنها بتستخدم قوسين عاديين ( ) مش أقواس مربعة [ ].
PYTHON — append() vs extend()
lst1 = [1, 2]
lst2 = [3, 4]

lst1.append(lst2)    # يضيف الـ list كعنصر واحد
print(lst1)           # [1, 2, [3, 4]]

lst1 = [1, 2]
lst1.extend(lst2)    # يضيف عناصر lst2 كلها
print(lst1)           # [1, 2, 3, 4] ✅
🚨 من امتحان 2025: لدمج عناصر list في list تانية، استخدم extend() — مش update() (ده للـ Set والـ Dict) ولا append() (ده يضيف كعنصر واحد كامل).

Set & Dictionary — المجموعات والقواميس

بيانات فريدة وبيانات بـ Key-Value Pairs

🎲 Set — المجموعة { }

الـ Set بتخزن قيم فريدة (مش بتكرر) وبدون ترتيب (Unordered) وبدون Index. يعني مش تقدر تقول s[0]!

PYTHON — Set
colors = {"red", "blue", "red", "green"}
print(colors)  # red واحدة بس!

# pop() في Set عشوائي!!
s = {1, 2, 3}
x = s.pop()   # مش عارف أيه هيتمسح!

# update() للـ Set: يضيف عناصر Set تانية
s1 = {1, 2}
s2 = {3, 4}
s1.update(s2)  # {1, 2, 3, 4}
🚨 من امتحانات 2021, 2025 (أسئلة متكررة):
• الـ Set هي Unordered AND Unindexed (مش Unordered AND Indexed)
• pop() في Set يمسح عنصر عشوائي — مش دايماً الأخير ولا الأول
• لدمج Set في Set تانية: استخدم update() (مش extend())
📖 Dictionary — { key: value }

الـ Dictionary بيخزن البيانات كأزواج من Key: Value. مناسب جداً لبيانات الـ Profile أو الـ Settings. الـ Keys لازم تكون فريدة.

PYTHON — Dictionary
student = {
    "name": "Laila",
    "age": 20,
    "dept": "IS"
}

print(student["name"])   # Laila

# تحديث قيمة
student["age"] = 21

# إضافة Key جديد
student["gpa"] = 3.9

print(student)
🚨 من امتحان 2021 — قاعدة مهمة:
لو ضفت Key موجود في الـ Dictionary، القيمة الجديدة بتكتب فوق القديمة بدون Error.
مثال: {"name": "Ahmed", "mobile": 123, "mobile": 987} → الـ mobile هيكون 987 (الأخيرة)
💡 y = d — Reference مش Copy: لما تكتب y = d مش بتعمل نسخة جديدة — الاتنين بيشاوروا على نفس الـ Dictionary. لو غيّرت في d، y هتتأثر برضو. عشان تعمل نسخة حقيقية استخدم d.copy() (بحرف سمول!)
🗃️ مقارنة هياكل البيانات — Data Structures Comparison
الهيكلالأقواسمرتبة؟لها Index؟تتعدل؟بتكرر؟استخدامها
list[ ]✅ نعم✅ نعم✅ نعم✅ نعمقوائم متغيرة بترتيب
tuple( )✅ نعم✅ نعم❌ لأ✅ نعمبيانات ثابتة (Immutable)
set{ }❌ لأ❌ لأ✅ نعم❌ لأقيم فريدة بدون تكرار
dict{ }✅ نعمبالـ Key✅ نعمKeys لأبيانات بتسمية (Key-Value)
💡 قاعدة الاختيار: بيانات بتتغير وبالترتيب؟ → List. ثابتة؟ → Tuple. بدون تكرار؟ → Set. كل قيمة ليها اسم (Label)؟ → Dictionary (مثلاً بيانات Student: name, age, dept).

Functions & Scope — الدوال ونطاق المتغيرات

كود قابل لإعادة الاستخدام — أساس Django Views

⚙️ تعريف الدوال — def

الـ Function بلوك كود بينفذ مهمة معينة ومش بيشتغل إلا لما تناديه (Call). بنعرفه بكلمة def. المهم تفهم الفرق بين تعريف الدالة واستدعاءها!

دالة بسيطة + Parameters

PYTHON
def greet(name):
    print(f"مرحباً {name}")

greet("Mona")
greet("Omar")

print vs return — الفرق المهم

PYTHON
def add(a, b):
    return a + b  # يرجع القيمة

result = add(5, 7)
print(result)   # 12
⚡ print vs return: print بيعرض للشاشة وبس. return بيرجع القيمة للبرنامج عشان تستخدمها بعدين. كتير من الـ Django Views بترجع قيم بـ return.
✨ *args و **kwargs
PYTHON
# *args: يجمع arguments في Tuple
def show(*args):
    print(args)

show(1, 2, 3)  # (1, 2, 3)

# **kwargs: يجمع keyword args في Dict
def person_info(**person):
    print("اسمه: " + person["name"])

person_info(name="Ahmed", age=10)
# يطبع: اسمه: Ahmed
# الـ age بيتجاهل لأن الكود مش محتاجه
📌 من امتحان 2021: **person بيجمع البيانات في Dictionary. الدالة بتطبع person["name"] بس — الـ age بيتجاهل بدون Error. ✅
λ Lambda Functions

الـ Lambda دالة سريعة بسطر واحد بدون اسم. فكرتها: lambda arguments: expression

PYTHON — Lambda
# سؤال بيتكرر في 2021 و2024!
y = lambda x: x / 20
print(y(100))   # 5.0 (مش 5!)
# القسمة بـ / دايماً float!

# دالة Lambda بدون Args
f = lambda: 10
print(f())   # 10
🚨 من امتحانات 2021, 2024: y = lambda x: x / 20 ثم y(100) = 5.0 وليس 5! دي من أكتر الأسئلة اللي بتتكرر.
🔭 Scope — نطاق المتغيرات

المتغير اللي بتعرّفه جوا Function هو Local Variable — بيعيش بس جوا الـ Function ومينقدرش تستخدمه بره منها.

PYTHON — Scope
def test():
    message = "أنا جوا الدالة"  # Local
    print(message)

test()
# print(message)  ← هيطلع NameError!

Modules, Errors & Files — التنظيم والأخطاء والملفات

Python بتنمو من Script واحد لمشروع ضخم

📦 Modules — الوحدات

الـ Module هو ملف Python بيحتوي كود قابل للاستيراد. Django نفسه مجموعة ضخمة من Modules!

PYTHON — Modules
# استيراد module جاهز
import math
print(math.sqrt(25))  # 5.0
print(math.pi)         # 3.14159...

# استيراد من module ملف ثاني
# helpers.py:
def greet(name): return f"Hello {name}"

# main.py:
from helpers import greet
print(greet("Nour"))
🐛 Error Types — أنواع الأخطاء

Q5 Final 2025 — T/F Q26
The output of the following statements is "Hello, My": 
b = "Hello, My Friend"
print(b[:8])
A
True
B
False
✅ الإجابة: A — True
الـ Slicing بالصيغة [:8] بتاخد من الأول (index 0) حتى index 7 (الرقم 8 نفسه مش متضمن). النص "Hello, My Friend" — الـ 8 حروف الأولى هي: H(0), e(1), l(2), l(3), o(4), ,(5), (6 space), M(7) = "Hello, M"... لحظة! نعيد العد:

انتبه: "Hello, My" فيها 9 حروف (شامل المسافة). لكن السؤال بيقول الناتج "Hello, My" — النتيجة الفعلية هي "Hello, M". لكن حسب إجابات 2025، الإجابة True. تأكد من السؤال في ورقة الامتحان.
