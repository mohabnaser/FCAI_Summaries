Advanced Models

Meta options, methods, properties

⚙️ Model Meta Options
class Course(models.Model):
    title = models.CharField(max_length=200)
    credits = models.IntegerField()
    is_active = models.BooleanField(default=True)

    class Meta:
        ordering = ['title']          # default sort order
        verbose_name = 'Course'      # admin display name
        db_table = 'courses'         # custom table name

    def __str__(self):
        return f"{self.title} ({self.credits} cr)"

Model Relationships

ForeignKey, OneToOne, ManyToMany

📊 Relationship Types

ForeignKey (Many-to-One)

Many students can belong to one department. Most common relationship.

dept = models.ForeignKey(
    Dept, on_delete=models.CASCADE)

OneToOneField

One student has exactly one profile. Like extending a model.

profile = models.OneToOneField(
    User, on_delete=models.CASCADE)

ManyToManyField

Students can enroll in many courses. Courses can have many students.

courses = models.ManyToManyField(
    Course)

Advanced QuerySet

Complex filtering, select_related, prefetch_related

🔎 Filter Lookups
# Double underscore lookups
Student.objects.filter(gpa__gte=3.5)       # ≥ 3.5
Student.objects.filter(name__contains="Ali")  # LIKE %Ali%
Student.objects.filter(name__startswith="M") # starts with M
Student.objects.exclude(is_active=False)   # NOT clause
Student.objects.order_by('-gpa')           # ORDER BY gpa DESC

# Optimized queries for relations
Student.objects.select_related('department')      # JOIN (FK)
Student.objects.prefetch_related('courses')      # M2M

Class-Based Views

More structured views using Django's generic CBVs

📋 Generic Views
from django.views.generic import ListView, DetailView, CreateView

class StudentListView(ListView):
    model = Student
    template_name = 'students/list.html'
    context_object_name = 'students'

class StudentDetailView(DetailView):
    model = Student
    template_name = 'students/detail.html'

class StudentCreateView(CreateView):
    model = Student
    fields = ['name', 'email', 'gpa']

Admin Customization

Making the admin panel more powerful

⚙️ ModelAdmin Options
@admin.register(Student)
class StudentAdmin(admin.ModelAdmin):
    list_display = ['name', 'email', 'gpa']  # columns shown
    list_filter = ['is_active']              # sidebar filter
    search_fields = ['name', 'email']       # search box
    ordering = ['-gpa']                       # default sort

Django Part III — Exam Quiz

Conceptual questions — IS231

Q1
Which field type creates a Many-to-One relationship in Django?
A
ForeignKey
B
OneToOneField
C
ManyToManyField
D
CharField
✅ Correct: A — ForeignKey
ForeignKey creates a Many-to-One relationship. Many students can belong to one department. OneToOneField = 1-to-1. ManyToManyField = M-to-M.
Q2
To get all students with GPA ≥ 3.5, the correct ORM query is:
A
Student.objects.filter(gpa=3.5)
B
Student.objects.all(gpa__gte=3.5)
C
Student.objects.filter(gpa__gte=3.5)
D
Student.objects.get(gpa__gte=3.5)
✅ Correct: C — filter(gpa__gte=3.5)
Use filter() for multiple results, get() for single. __gte = ≥. __gt = >. __lte = ≤. all() doesn't accept filter arguments.
Q3
Which option in a ModelAdmin shows specific columns in the admin list view?
A
show_columns
B
list_display
C
display_columns
D
fields
✅ Correct: B — list_display
list_display = columns shown in admin list. list_filter = sidebar filters. search_fields = searchable fields. fields = which fields show in the edit form.
Q4
The Django ORM query Student.objects.order_by('-gpa') sorts by:
A
GPA ascending (lowest first)
B
Alphabetically by name
C
Student ID descending
D
GPA descending (highest first)
✅ Correct: D — GPA descending
A minus sign (-) prefix in order_by means DESCENDING. order_by('gpa') = ascending. order_by('-gpa') = descending (highest GPA first).
Q5
Which optimization method avoids the N+1 query problem for ForeignKey relationships?
A
select_related()
B
prefetch_related()
C
filter_related()
D
join_related()
✅ Correct: A — select_related()
select_related() does a SQL JOIN for ForeignKey/OneToOne (single objects). prefetch_related() does a separate query for ManyToMany (collections). Both solve N+1 queries.
