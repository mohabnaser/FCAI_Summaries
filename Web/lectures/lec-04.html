Web Flow — دورة حياة الويب

من أول الكليك لحد ظهور الصفحة

🔄 Client vs Server (العميل والخادم)

الإنترنت عبارة عن حوار مستمر بين طرفين:

  • Client (العميل): المتصفح أو تطبيق الموبايل، هو اللي بيطلب البيانات. وظيفته يعرض التصميم (HTML/CSS/JS) ويجمع بيانات المستخدم.
  • Server (الخادم): الجهاز اللي بيستقبل الطلب، بيعالجه، وبيرد بالإجابة. ده المكان اللي بيشتغل فيه الـ Backend.
🏗️ Frontend vs Backend vs Database

🎨 Frontend

الجزء اللي اليوزر بيشوفه ويتعامل معاه. (أزرار، ألوان، فورمة).

⚙️ Backend

الجزء الخفي! ده اللي بيشغل المنطق (Business Logic)، بيتأكد من الصلاحيات والـ Validation.

💾 Database

الذاكرة الدائمة للمشروع. بتحفظ بيانات اليوزرز والكورسات عشان مانفقدهاش.

🚀 رحلة الـ Request (طلب الويب)

لما بتكتب لينك أو تدوس على زرار، المتصفح بيبعت Request للسيرفر، السيرفر بيعالجه (وممكن يكلم الداتابيز)، وبعدين يبعت Response (الرد) للمتصفح عشان يعرض الصفحة.

🔗

URLs — الروابط وتشريحها

عنوان الويب بيتكون من إيه؟

🔍 تشريح الرابط (URL Parts)

أي لينك (Uniform Resource Locator) متقسم لأجزاء مهمة جداً:

https://portal.cu.edu.eg:443/students/profile?id=25

https          -> Protocol (البروتوكول الآمن)
portal.cu.edu.eg -> Domain (اسم الموقع / Host)
:443           -> Port (البورت الاختياري اللي بينفصل بنقطتين)
/students/profile -> Path (المسار أو الـ Route)
?id=25         -> Query String (البارامترات الإضافية)
موضع امتحان 2021: سأل عن الجزء اللي بيكون "اختياري" وممكن يتكتب بعد الـ Host ومفصول بنقطتين `colon`... الإجابة كانت Port!
📍 Path vs Query String
  • Path: بيحدد المورد الأساسي (إحنا فين بالظبط). زي /courses/list.
  • Query String: بيبدأ بعلامة استفهام ?، وبيستخدم للفلترة والبحث والتفاصيل الإضافية. زي ?level=2&sort=name.

HTTP & Methods — لغة التخاطب وطرق الطلب

السيرفر والعميل بيتفاهموا إزاي؟

💬 إيه هو الـ HTTP؟

الـ HTTP (HyperText Transfer Protocol) هو لغة التخاطب الأساسية للويب. بينظم شكل الـ Request (الطلب) والـ Response (الرد).

Request (من العميل)

لازم دايماً يحتوي على: Request Line (زي Method والمسار) و Headers (البيانات الوصفية). وممكن يحتوي على Body (لو بنبعت داتا زي الفورمة).

Response (من السيرفر)

لازم يحتوي على: Status Code (كود الحالة زي 200 أو 404)، و Headers، وغالباً Body (الكود أو الداتا اللي هترجع).

🛠️ طرق الطلب (HTTP Methods)
  • GET: للقراءة فقط (Safe). بيجيب صفحة أو بيانات، والبيانات اللي بتتبعت معاه بتظهر في الـ URL.
  • POST: للإرسال والحفظ. بيستخدم لإنشاء داتا جديدة (زي التسجيل أو اللوجين). بيحط الداتا في الـ Body فبيكون أكثر أماناً من الـ GET.
  • PUT / PATCH: للتعديل والتحديث. الـ PUT بيعمل تعديل كامل للـ Object، بينما الـ PATCH بيعمل تحديث جزئي (Partial Update).
  • DELETE: للحذف. المفروض يكون Idempotent (يعني لو كررته ميعملش مشكلة).
موضع امتحان 2021 & 2025:
1. الـ POST أكثر أماناً من הـ GET. (امتحان 2021)
2. الـ PATCH هو اللي بيعمل التعديل الجزئي (Partial Update)، مش הـ PUT! (فخ امتحان 2025).
🛡️ Backend Validation — ليه بنعمل فحص في السيرفر؟

الفحص اللي بيتعمل في المتصفح (Frontend Validation) غرضه يسهل على اليوزر، بس مينفعش نثق فيه أبداً! الهاكر يقدر يغير كود الـ HTML بسهولة، عشان كده الـ Backend لازم يفحص الداتا تاني ويتأكد إنها صح وآمنة.

JSON & APIs — نقل البيانات بسلاسة

لغة التواصل الحديثة في الويب

🧩 إيه هو الـ JSON؟

بدل ما السيرفر يرجع صفحة HTML كاملة، الـ APIs الحديثة بترجع بيانات خام بصيغة JSON (JavaScript Object Notation). دي أسهل طريقة عشان تطبيقات الموبايل والمواقع الحديثة تفهم الداتا وتعرضها.

# مثال لـ JSON
{
    "status": "success",
    "user_id": 25,
    "role": "student"
}
موضع امتحان 2025: نوع المحتوى في الرد (Response Content Type) هو اللي بيحدد المتصفح هيتعامل مع الرد إزاي (هل هو JSON ولا صورة ولا HTML)، وده بيتسمى الـ Media Type أو MIME Type.
🏠 Localhost:8000

لما بنشتغل في مرحلة التطوير (Development) بنستخدم localhost يعني "جهازي أنا". والـ 8000 هو الـ Port الافتراضي اللي Django بيشتغل عليه.

Status Codes — أكواد الحالة

السيرفر بيرد ويقول إيه الأخبار؟

🔢 عائلات الـ Status Codes

2xx: Success (نجاح)

  • 200 OK: كل حاجة تمام وتم بنجاح!
  • 201 Created: تم إنشاء العنصر بنجاح (زي لما تعمل حساب جديد).
  • 204 No Content: نجاح بس مفيش داتا رجعت في الـ Body.

3xx: Redirection (توجيه)

  • 301 Moved Permanently: الصفحة اتنقلت لمكان تاني بشكل دائم.
  • 302 Found (Temporary): توجيه مؤقت. (شائع جداً بعد عمل Login ناجح عشان يحولك للوحة التحكم).

4xx: Client Error (خطأ من العميل)

  • 400 Bad Request: طلب مش مفهوم أو ناقص.
  • 401 Unauthorized: إنت مش عامل Login (السيرفر بيقولك "إنت مين؟").
  • 403 Forbidden: إنت عامل Login بس معكش صلاحية (السيرفر بيقولك "أنا عارفك بس مش هسمحلك").
  • 404 Not Found: الصفحة أو المسار مش موجود.

5xx: Server Error (خطأ في السيرفر)

  • 500 Internal Server Error: الكود ضرب جوه السيرفر! العميل ملوش ذنب.
  • 503 Service Unavailable: السيرفر واقع أو عليه ضغط ومش قادر يرد.
🤔 404 ولا 500؟ (سؤال الـ Debugging)

لو المسار اللي كتبته مش موجود أصلاً، هيظهرلك 404. لكن لو المسار صح بس الكود بتاع بايثون ضرب جواه Error، السيرفر هيرد بـ 500.

Web Basics — Final 2021

أسئلة من فاينال 2021 عن أساسيات الويب والـ HTTP 🇪🇬

Q1 Final 2021
Find the oddly matched HTTP status codes:
A
200 - OK
B
400 – Bad Request
C
301 – Moved Permanently
D
304 – Not Found
✅ الإجابة الصح: D — 304 – Not Found
الـ "Not Found" كودها الشهير 404 مش 304. الـ 304 دي بتاعت הـ Not Modified، فدي التوصيلة الغلط في الاختيارات.
Q2 Final 2021
The first line in the request message is called the ______ line, and the first line in the response message is called the ______ line.
A
Request, Status
B
Status, Status
C
Response, Status
D
None of the Above
✅ الإجابة الصح: A — Request, Status
رسالة الطلب بتبدأ بـ Request Line (زي GET /path HTTP/1.1). أما رسالة الرد بتبدأ بـ Status Line (زي HTTP/1.1 200 OK).
Q3 Final 2021
An HTTP request message always contains ______.
A
A Header and A Body
B
A Request Line and A Header
C
A Status Line, A Header and A Body
D
None of the Above
✅ الإجابة الصح: B — A Request Line and A Header
الـ Body مش إجباري دايماً (الـ GET مثلاً مابيكونش فيه Body). لكن أي Request لازم يكون فيه Request Line و Headers.
Q4 Final 2021
______ is more robust and secure than ______.
A
Get, Post
B
Post, Put
C
Post, Get
D
Get, Put
✅ الإجابة الصح: C — Post, Get
الـ POST بيبعت الداتا متخبية جوه الـ Body بتاع הـ Request، لكن הـ GET بيبعتها مكشوفة فوق في الـ URL، فالـ POST أأمن.
Q5 Final 2021
In a URL, an optional ______ can be inserted between the host and the path, separated from the host by a colon.
A
Port
B
Protocol
C
Query string
D
Parameter
✅ الإجابة الصح: A — Port
اللي بييجي بعد الـ Domain وبيبقى مفصول بنقطتين : هو الـ Port. زي localhost:8000.
Q6 Final 2021
An HTTP Response message must contain ______.
A
Form data
B
Query string
C
Status code
D
HTML Body
✅ الإجابة الصح: C — Status code
أي رد (Response) بييجي من السيرفر لازم يكون فيه كود يقولك إيه الأخبار (Status Code) زي 200 أو 404.

Web Basics — Final 2025

أسئلة الويب من أحدث امتحان 🇪🇬

Q1 Final 2025 (T/F)
We use the put method in the HTTP request message for doing partial updates.
A
True
B
False
✅ الإجابة الصح: B — False
الـ PUT بيعمل Full Update (تعديل كامل للعنصر). لكن التعديل الجزئي (Partial Update) بيتعمل بـ PATCH!
Q2 Final 2025
An HTTP response content type denotes:
A
The media type of the content
B
The image size of the content
C
The text size of the content
D
None of the answers!
✅ الإجابة الصح: A — The media type of the content
الـ Content-Type (أو الـ MIME type) بيعرف المتصفح نوع الداتا اللي جاية: هل هي HTML ولا JSON ولا صورة. ملوش دعوة بالحجم خالص!
Q3 Final 2025
Using authorization, having a response of status 200 means that:
A
User is not authorized to see the response
B
User is authorized to see the response
C
Content cannot be displayed after authorization
D
None of the answers!
✅ الإجابة الصح: B — User is authorized
كود 200 يعنى OK، يعني إنت مصرح ليك تشوف الداتا والدنيا تمام. (اتكرر نفس السؤال بالظبط في فاينال 2021).
Q4 Final 2025
Using authorization, having a response of status 403 means that:
A
Server is reachable but the user is not authorized to see the response
B
Server is not reachable, and the user is not authorized to see the response
C
Server is reachable and no content available to show the user
D
None of the answers!
✅ الإجابة الصح: A — Server is reachable but the user is not authorized
الـ 403 Forbidden معناها السيرفر شغال ووصلنا له تمام (Reachable)، بس إنت معكش "صلاحية" (Not Authorized) تدخل الصفحة دي بالذات (مثلاً صفحة أدمن وإنت يوزر عادي).
