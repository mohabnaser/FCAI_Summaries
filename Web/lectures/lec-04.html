Client & Server Model

Every web interaction has two sides

💻 What Happens When You Open a Website?
1
User types URL or clicks link
Browser prepares an HTTP request.
2
HTTP Request sent over network
Uses TCP protocol on port 80 (HTTP) or 443 (HTTPS).
3
Server processes the request
Runs logic, queries database, applies permissions.
4
Server sends HTTP Response
Contains a status code, headers, and (usually) HTML/JSON body.
5
Browser renders the page
Displays the received content to the user.
⚙️ Frontend vs Backend vs Database

🎨 Frontend

What the user sees. HTML, CSS, JavaScript. Runs in the browser on the client's machine.

⚙️ Backend

Logic, processing, permissions. Runs on the server. Django, Python, etc.

🗄️ Database

Persistent data storage. Only the backend talks to it directly — not the browser.

🔗

What is a URL?

Uniform Resource Locator — the address of every web resource

🏗️ Parts of a URL

Example: https://portal.cu.edu.eg:443/students/profile?id=25

Protocol

https — Secured. http — Unsecured.

Domain / Host

portal.cu.edu.eg — The server's address.

Port (Optional)

443 — HTTP default=80, HTTPS default=443. Separated from host by a colon.

Path

/students/profile — Which resource/route on the server.

Query String

?id=25 — Extra parameters/filters. Multiple params: &.

Fragment

#section — Client-side anchor. NOT sent to the server.

Exam Tip: "An optional port number can be inserted between the host and the path, separated from the host by a colon."
📡

HTTP Protocol

HyperText Transfer Protocol — the language of the web

📋 HTTP Basics
  • HTTP = HyperText Transfer Protocol.
  • Defines the format of requests and responses between client and server.
  • Runs on TCP port 80. HTTPS (secure) runs on TCP port 443.
  • Does not decide business logic — just the communication format.
Exam 2021: HTTP uses the services of TCP on well-known port 80.
📦 Headers vs Body

Headers

Metadata about the request/response. Content type, auth tokens, cookies, cache info. Present in both requests AND responses.

Body

The actual data. GET has no body. POST/PUT carry data in the body. Response body = the HTML/JSON page content.

Exam 2021: The body of the request can contain parameters sent to the server.
⚡

HTTP Methods

The method tells the server what you want to do

📋 CRUD Methods
MethodActionHas Body?Use Case
GETRead data❌ No bodyView page, search, get list
POSTCreate / Send data✅ Has bodyLogin form, registration, create record
PUTFull update✅ Has bodyReplace an existing resource entirely
PATCHPartial update✅ Has bodyUpdate only one field
DELETERemoveOptionalDelete a record
Exam 2021: POST is more robust and secure than GET — GET data is visible in the URL. POST data is in the body (hidden from URL bar).
📤

HTTP Request Structure

What the browser sends to the server

📨 Request Format

Structure:

[method] [URL] [version]
[headers]

[body — POST/PUT only]

Example GET:

GET http://www.google.com HTTP/1.1
Host: www.google.com
User-Agent: Mozilla/5.0
Accept: text/html
Exam 2021: An HTTP request message always contains a Request Line and a Header. A body is optional (only POST/PUT).
📋 Common Request Headers
HeaderMeaning
RefererURL the request came from
User-AgentBrowser making the request
AcceptContent types the client accepts
CookieStored cookies from previous server responses
If-Modified-SinceCaching — last time resource was fetched
📥

HTTP Response Structure

What the server sends back

📩 Response Format

Structure:

[version] [status code] [reason]
[headers]

[body — the page/data]

Example:

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 17151

<html>...</html>
Exam 2021: In the response, the first line is called the Status Line. Every HTTP response MUST contain a status code.
📋 Key Response Headers
HeaderMeaning
Content-TypeType of content (text/html, application/json)
Content-LengthBody size in bytes
ExpiresTime-based caching — when the resource expires
LocationRedirect target URL
Set-CookieCreates a cookie in the browser
Exam 2021: Expires: Tuesday, 17 Dec 2020... is a Response header denoting Time-Based Caching.
🚦

HTTP Status Codes

The server's 3-digit summary of what happened

📊 Status Code Families

2xx — Success ✅

Request was received, understood, and processed.

3xx — Redirection 🔀

Go to a different URL. Common after login or form submit.

4xx — Client Error ❌

The request has a problem. Bad syntax, unauthorized, not found.

5xx — Server Error 💥

The server failed to handle a valid request. Backend bug/crash.

CodeNameMeaning
200OK✅ Request succeeded
201Created✅ New resource created
301Moved Permanently🔀 Resource permanently at new URL
302Found (Redirect)🔀 Temporary redirect (login → dashboard)
304Not Modified💾 Use cached version — resource unchanged
400Bad Request❌ Malformed request from client
401Unauthorized🔑 Need to authenticate (not logged in)
403Forbidden🚫 Logged in but no permission
404Not Found🔍 Resource doesn't exist
500Internal Server Error💥 Backend bug/crash
Classic Trap — 304 vs 404:
304 = "Not Modified" (3xx = Redirection family) → Use cache ✅
404 = "Not Found" (4xx = Client Error family) → Resource missing ❌
"304 = Not Found" is the WRONG match! It's "304 = Not Modified".
401 vs 403: 401 = not logged in yet. 403 = logged in but not allowed.
🍪

Cookies & Security

Sessions, auth, authorization, HTTPS

🍪 Cookies
  • Created by the Server — sent via Set-Cookie response header.
  • Stored by the browser — sent back in every future request via Cookie header.
  • Used for sessions, preferences, tracking, login state.
Exam 2021: A cookie is created by the Server.
🔐 Authentication vs Authorization

🔑 Authentication

Who are you? — Verify identity. Login with username/password. 401 = not authenticated.

🛡️ Authorization

What can you do? — Check permissions after login. 403 = no permission. 200 = authorized, content shown.

Exam 2021: To denote permission of using a resource → Authorization.
🔒 HTTPS & TLS
  • HTTPS = HTTP over TLS (Transport Layer Security).
  • A lock icon in the browser = HTTPS over TLS = secured connection.
  • Data is encrypted — prevents eavesdropping. Runs on port 443.
Exam 2021: A lock icon in the browser denotes "Secured Hypertext Transfer Protocol over TLS, secured".

HTTP & URLs — Final 2021 (14 Questions)

All real exam questions with Egyptian slang explanations 🇪🇬

Q1 Final 2021
A web page has 4 JPEG images and HTML text. The total number of objects is:
A
1
B
3
C
5
D
8
✅ Correct: C — 5
1 HTML file + 4 JPEG images = 5 objects total. Each file is counted as a separate HTTP object. كل ملف بيتحسب object لوحده. HTML واحدة = 1 object، وكل صورة = 1 object. يعني 1 + 4 = 5 objects.
Q2 Final 2021
Which is the oddly matched HTTP status code?
A
200 — OK
B
400 — Bad Request
C
301 — Moved Permanently
D
304 — Not Found
✅ Correct: D — 304 — Not Found is WRONG
304 means "Not Modified" (use cached version). 404 = Not Found. This is a classic trick question! ده أشهر فخ في الامتحان! 304 = Not Modified (استخدم الـ cache). 404 = Not Found (مش موجود). لو شفت 304 — Not Found اعرف إنه غلط على طول.
Q3 Final 2021
HTTP uses the services of ______ on well-known port 80:
A
TCP
B
UDP
C
SMTP
D
HTTPS
✅ Correct: A — TCP
HTTP runs over TCP on port 80. HTTPS runs over TCP/TLS on port 443. HTTP → TCP port 80. HTTPS → TCP/TLS port 443. احفظ الأرقام دي كويس، بتيجي كتير في الامتحان.
Q4 Final 2021
In the HTTP response message, the first line is called the ______ line:
A
Status
B
Request
C
Response
D
Header
✅ Correct: A — Status Line
First line of response = Status Line (e.g. HTTP/1.1 200 OK). First line of request = Request Line. Response: أول سطر = Status Line. Request: أول سطر = Request Line. لو السؤال عن الـ response قول Status، لو عن الـ request قول Request Line.
Q5 Final 2021
An HTTP request message always contains ______:
A
A Header and A Body
B
A Request Line and A Header
C
A Status Line, A Header and A Body
D
None of The Above
✅ Correct: B — Request Line + Header
Every request has a Request Line (method + URL + version) and headers. Body is optional — GET has no body. الـ request دايماً فيه Request Line (أول سطر) وـ Headers. الـ Body اختياري — GET مالوش body، بس POST وـ PUT عندهم body. وبرضو لو سألك عن Response: Status Line + Headers + (Body اختياري).
Q6 Final 2021
______ is more robust and secure than ______:
A
GET, POST
B
POST, PUT
C
POST, GET
D
None of The Above
✅ Correct: C — POST is more secure than GET
POST sends data in the body (not visible). GET puts data in the URL (visible in browser history and logs). GET بيبعت الداتا في الـ URL — يعني أي حد يشوف الشاشة يشوفها. POST بيبعت الداتا في الـ body — أأمن بكتير. عشان كده forms الـ Login بتستخدم POST مش GET.
Q7 Final 2021
In a URL, an optional ______ can be inserted between the host and the path, separated from the host by a colon:
A
Port Number
B
Protocol
C
Host
D
None of Them
✅ Correct: A — Port Number
URL format: https://host:port/path. The port is optional and separated by a colon. الـ port بيجي بعد الـ host بعلامة colon (:). زي portal.cu.edu.eg:443. لو مش محدد، HTTP بياخد 80 وـ HTTPS بياخد 443 تلقائي.
Q8 Final 2021
A cookie is created by ______:
A
Server
B
Client
C
Host
D
None of Them
✅ Correct: A — Server
The server creates cookies via Set-Cookie response header. The browser stores and returns them. الـ cookie بيتعملها الـ Server وبيبعتها في الـ response header. البراوزر بيحفظها وبيبعتها مع كل request بعد كده. مش البراوزر اللي بيعملها — السيرفر!
Q9 Final 2021
The ______ of the ______ can contain parameters sent to the server:
A
Header, response
B
Header, request
C
Body, response
D
Body, request
✅ Correct: D — Body of the Request
POST request body contains the parameters sent to the server. GET puts params in the URL query string. البيانات اللي بتتبعت للـ server بتيجي في الـ body من الـ request (مش response). ده بيحصل في POST/PUT — الـ body هو اللي بيشيل الـ form data.
Q10 Final 2021
An HTTP Response message must contain ______:
A
Content coming from the server
B
Content coming from the client
C
Status code
D
None of them
✅ Correct: C — Status code
Every HTTP response MUST include a status code (200, 404, 500…). The body is optional (204 = success but no body). الـ response لازم يكون فيه status code دايماً. الـ body ممكن يكون فاضي (زي 204 No Content). يعني اللي إلزامي في الـ response: Status Line + Headers + Status Code. الـ body اختياري.
Q11 Final 2021
"Expires: Tuesday, 17 Dec 2020 18:30 GMT" is a ______ header denoting ______:
A
Request, Time-Based Caching
B
Response, Content-Based Caching
C
Request, Content-Type Caching
D
Response, Time-Based Caching
✅ Correct: D — Response + Time-Based Caching
Expires header = Response header. It tells the browser when the cached resource expires. This is time-based caching. Expires header ده response header. بيقول للبراوزر "الـ cache ده صلاحيته بتخلص يوم كذا". ده بيتسمى Time-Based Caching لأنه بيعتمد على تاريخ.
Q12 Final 2021
To denote permission of using a resource, you must use ______:
A
Authentication
B
Validation
C
Authorization
D
None of Them
✅ Correct: C — Authorization
Authentication = who are you? Authorization = what are you allowed to do? Permission = Authorization. Authentication = "إيه اسمك ومين انت؟" (Login). Authorization = "عندك إذن تعمل ده؟" (Permissions). Permission = Authorization. الفرق ده كلاسيك في الامتحانات.
Q13 Final 2021
Using authorization, a response with status 200 means that ______:
A
The user is not authorized
B
Content cannot be displayed
C
None of the answers
D
User is authorized to see the response
✅ Correct: D — User is authorized
200 OK = success. In auth context: user has permission and the content is returned successfully. 200 = OK = كل حاجة تمام. في سياق الـ authorization: المستخدم معاه الـ permission وشاف الـ content. 403 = متاله حساب بس ملوش permission. 401 = ملوش حساب خالص.
Q14 Final 2021
A lock icon in the browser denotes ______ for ______ connection:
A
HTTP over TCP, secured
B
HTTPS over TLS, secured
C
HTTP over TCP, unsecured
D
None of the answers
✅ Correct: B — HTTPS over TLS, secured
Lock icon = HTTPS (HTTP Secure) which runs over TLS. Connection is encrypted. أيقونة القفل = الموقع بيستخدم HTTPS = بيشتغل فوق TLS = الـ connection مشفر وآمن. لو مفيش قفل → HTTP عادي مش مشفر.
