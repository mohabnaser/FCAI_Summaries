Variables & Scope

var vs let vs const

🔢 Variable Declarations
KeywordScopeHoisted?Reassignable?Re-declarable?
varFunction scope✅ Yes (as undefined)✅ Yes✅ Yes
letBlock scope {}⚠️ (TDZ)✅ Yes❌ No
constBlock scope {}⚠️ (TDZ)❌ No❌ No
Exam Traps:
• const must be initialized when declared — you can't do const x; then assign later.
• let is block-scoped — not accessible outside {}.
• const cannot be overwritten (reassigned) after declaration.

Types & Type Coercion

JavaScript converts types automatically

⚡ Type Coercion Rules
// String + Number → String concatenation
"40" + 6          // → "406"  (string concat)
"20" + 10         // → "2010" (string concat)

// String - Number → Number subtraction
"20" - 10         // → 10     (numeric subtraction)

// == vs ===
200 == "200"      // → true  (coerces types)
200 === "200"     // → false (strict: different types)
Exam Tip: Addition with a string = string concatenation. Subtraction with a string = numeric. Evaluate expressions left to right.
⚠️ JavaScript Error Types
  • Syntax Error — missing brackets, wrong syntax
  • Missing brackets/semicolons — some JS engines handle missing semicolons (ASI)
  • Division by zero — NOT an error in JavaScript! Returns Infinity
Exam Trap: In JavaScript, division by zero is NOT an error — it returns Infinity. This is different from Python where it throws ZeroDivisionError.

Functions

Regular, arrow, and rest parameters

→ Arrow Functions
// Arrow function (concise syntax)
const f = (a, b) => a + b + 5;
f(3, 4)   // → 12  (3 + 4 + 5 = 12)
f(4, 5)   // → 14  (4 + 5 + 5 = 14)

// Regular function expression
const g = function() { return 10; };
... Rest Parameters
function multiply(...myNumbers) {
  let total = 1;
  for (let n of myNumbers) total *= n;
  return total;
}
Exam: The ... (rest parameter) allows the function to change the arguments into an array. It is called the Rest Parameter.

DOM Manipulation

Accessing and changing HTML from JavaScript

🔍 Common DOM Methods
// Select element by ID
document.getElementById("myElement").innerHTML = "CS 105";

// Change content of a paragraph with id="demo"
document.getElementById("demo").innerHTML = "Hello World!";

// Access text content (text node)
document.getElementById("id1").firstChild.nodeValue;
Exam: How are objects organized in the HTML DOM? → Hierarchy (tree structure). The purpose of form control elements? → User Interface elements.
💻 JavaScript Execution
  • JavaScript runs on the user's machine in the web browser (client-side).
  • The web server does NOT execute JavaScript (usually).
  • JavaScript is dynamically typed — variable types determined at runtime.

Scope & Common Patterns

Understanding var/let/const in real code

🔍 eval() Function
var writeMe = "console.log('document.writeln(x);')";
var x = 13;
eval(writeMe);
// eval() executes the STRING as code
// The string contains: console.log('document.writeln(x);')
// Output: prints "document.writeln(x);" in the console
Exam: eval() treats the string as code and runs it. The string here calls console.log with the literal text 'document.writeln(x);' — so it prints that text, NOT the value 13.
📊 JS Output Snippet
x = "20";
y = 10;
document.writeln(x + y, x - y);
// x + y → "20" + 10 → "2010"  (string concat)
// x - y → "20" - 10 → 10     (numeric sub)
// Output: 2010 10

Q1 Final 2021
When a user views a page containing a JavaScript program, which machine executes the script?
A
The User's machine running a Web browser
B
The Web server
C
Both client and server
D
None of the above
✅ Correct: A — The User's machine (Web browser)
JavaScript is a client-side scripting language. It runs in the browser on the user's machine, NOT on the web server.
Q2 Final 2021
Which of the following lines of JavaScript code will change the HTML inside an element with id="myElement" to "CS 105"?
A
document.getElementById("myElement").innerHTML = "CS 105";
B
html["myElement"] = "CS 105";
C
html["myElement"].innerHTML = "CS 105";
D
document.html["myElement"].innerHTML = "CS 105";
✅ Correct: A — document.getElementById("myElement").innerHTML
The correct DOM API: use document.getElementById() to select an element by its id, then set .innerHTML to change its content.

A

B

C

✅ Correct: C — Division by zero
In JavaScript, dividing by zero returns Infinity (not an error). Also, some JS engines handle missing semicolons via Automatic Semicolon Insertion (ASI).

The DOM is organized as a tree/hierarchy of nodes. The document is the root, then html, then head/body, then their children.
