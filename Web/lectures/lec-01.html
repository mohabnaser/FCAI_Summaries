What is Backend Development? — إيه هو الـ Backend؟

العقل المدبر المخفي وراء أي موقع ويب

💡 تعريف الـ Backend

الـ Backend هو الجزء اللي بيشتغل على الـ Server side (ناحية الخادم). المستخدم مش بيشوفه مباشرة، لكنه بيمثل "العقل" الفعلي للبرنامج. هو اللي بيستقبل الـ Requests، وينفذ الـ Logic، ويتعامل مع قواعد البيانات (Databases)، وبيطبق الحماية والأمان (Security)، وبيرجع الـ Responses للـ Browser.

⚙️ مسؤوليات الـ Backend الرئيسية

البيانات والمنطق (Data & Logic)

  • تخزين وجلب البيانات من الـ Database.
  • التحقق من صحة مدخلات المستخدم (Input Validation) قبل حفظها.
  • تطبيق قواعد العمل (Business Rules) زي عمليات الـ Login وحساب التقديرات الدراسية.
  • إنشاء محتوى ديناميكي (Dynamic Content) بناءً على بيانات المستخدم الحالي.

الأمان والتكامل (Security & Integration)

  • التحقق من هوية المستخدمين (Authentication) وإدارة الصلاحيات (Permissions).
  • التعامل مع الأخطاء (Error Handling) وإرسال رسايل مناسبة بدل ما السيستم يقع.
  • التواصل مع الخدمات الخارجية (External APIs) زي بوابات الدفع الإلكتروني أو إرسال الإيميلات.
  • توفير البيانات للموقع أو لتطبيقات الموبايل.
🎯 أمثلة من حياتنا اليومية لعمل الـ Backend
  • عملية تسجيل الدخول (Login) للـ account بتاعك.
  • البحث عن منتج معين في موقع تسوق وعرض النتايج المتاحة بس.
  • تسجيل البيانات في فورم التقديم للكلية وحفظها في قاعدة البيانات.
  • التحقق إذا كان الـ Username اللي بتختاره متوفر ولا حد تاني حجزه قبلك.
  • عرض لوحة البيانات (Personalized Dashboard) الخاصة بتقديراتك ودرجاتك أنت بس.

Web Architecture & 3-Tier Model — بنية الويب وموديل الـ 3-Tier

رحلة البيانات من المتصفح إلى السيرفر والـ Database

🌐 كيف يعمل موقع الويب؟ (خطوة بخطوة)
1
خطوة المستخدم (User Action)
المستخدم بيكتب URL في الـ Browser أو بيدوس على Link.
2
إرسال الـ HTTP Request
الـ Browser بيبعت طلب (HTTP Request) للسيرفر عبر شبكة الإنترنت.
3
معالجة الـ Backend
الـ Backend بيستقبل الـ Request ويبدأ يفحصه ويشوف محتاج ينفذ إيه (زي التحقق من الصلاحيات).
4
التعامل مع الـ Database
الـ Backend بيكلم الـ Database عشان يقرا منها بيانات أو يكتب فيها بيانات جديدة.
5
إرسال الـ HTTP Response
السيرفر بيرجع الرد (HTTP Response) للـ Browser وفيه الكود (HTML/JSON) والـ Status Code.
6
عرض المتصفح (Render)
الـ Browser بيستلم الرد ويعرضه بشكل مرئي ومفهوم للمستخدم.
📊 نموذج الطبقات الثلاث (3-Tier Application Model)

1. Presentation Layer (Frontend)

الواجهة اللي المستخدم بيتعامل معاها مباشرة. بتتكون من HTML, CSS, JavaScript وبترن بالكامل جوة الـ Browser.

2. Application Layer (Backend)

المنطق والعمليات الحسابية وقواعد السيستم. هنا بيشتغل كود الـ Python والـ Django على الـ Server.

3. Data Layer (Database)

مكان حفظ البيانات بشكل دائم. زي SQLite, MySQL, PostgreSQL.

⚠️ قاعدة ذهبية للامتحان: الـ Database لا تتواصل أبداً مع المستخدم مباشرة! الـ Backend (Application Layer) هو دايماً وسيط النص اللي بيكلم الـ Database وبيجهز البيانات للـ Frontend.

Request & Response Cycle — دورة الطلب والاستجابة

كل حركة على الويب هي عبارة عن سؤال وجواب

📤 مكونات الـ HTTP Request

لما الـ Browser بيبعت Request للسيرفر، بيبعت معاه شوية تفاصيل مهمة:

  • HTTP Method: نوع العملية المطلوبة (زي GET لجلب البيانات، أو POST لإرسال وحفظ بيانات جديدة).
  • URL Path: المسار أو الصفحة المطلوبة على السيرفر زي /login أو /students.
  • Headers: معلومات إضافية (Metadata) زي نوع المتصفح ولغة المستخدم والـ Caching.
  • Body: البيانات الفعلية المبعوتة (ودي اختيارية وبتكون موجودة في POST أو PUT زي بيانات الفورم أو ملفات مرفوعة).
📥 مكونات الـ HTTP Response

الرد اللي بيرجع من السيرفر برضه بيتكون من أجزاء محددة:

  • Status Code: رقم من 3 خانات بيعبر عن نتيجة الطلب (زي 200 للموافقة، أو 404 لو الصفحة مش موجودة، أو 500 لو فيه مشكلة في كود السيرفر).
  • Headers: معلومات إضافية السيرفر بيبعتها زي نوع المحتوى المرجّع (HTML أو JSON) وتاريخ انتهاء الصلاحية.
  • Response Body: المحتوى الفعلي اللي هيتعرض للمستخدم (زي صفحة HTML أو بيانات JSON).
⚡ مقارنة هامة جداً: GET مقابل POST
الصفة GET Method POST Method
الهدف طلب وقراءة البيانات بدون تعديل. إرسال وحفظ بيانات جديدة أو إحداث تغيير.
البيانات بتتبعت في الـ URL كـ Query Parameters. بتتبعت بشكل مخفي جوة الـ Request Body.
الأمان غير آمنة للبيانات الحساسة (لأن البيانات بتظهر في الـ URL بار وتتحفظ في الـ History). أكثر أماناً وحماية لأن البيانات مخفية عن الـ URL بار.
الـ Body لا تحتوي على Body. تحتوي على Request Body يحمل البيانات.
💡 موضع سؤال امتحان: "POST is more robust and secure than GET" (الـ POST أكثر قوة وأماناً من الـ GET) لأن الداتا مش بتظهر مكشوفة في الـ URL.
🔗 Anatomy of a URL — تشريح الـ URL

الـ URL هو العنوان الفريد لأي صفحة أو ملف على الويب. بيتكون من أجزاء:

https://example.com:80/students/list/?level=2
|___|  |_________| |__| |___________| |_____|
  1         2       3         4          5
  1. Protocol: زي https (الآمن) أو http (العادي).
  2. Domain / Host: اسم السيرفر أو الموقع (زي example.com).
  3. Port (اختياري): رقم المنفذ. الـ default في الـ HTTP هو 80 والـ HTTPS هو 443. بيفصل بينه وبين الـ Host بنقطتين فوق بعض :.
  4. Path: مسار الصفحة على السيرفر (زي /students/list/).
  5. Query String: المعاملات الإضافية للفلترة وتبدأ بـ ? (زي level=2).
💡 نقطة امتحان سابقة: "In a URL, an optional port number can be inserted between the host and the path, separated from the host by a colon."

Static vs Dynamic Websites — الفرق بين المواقع الـ Static والـ Dynamic

ليه بنحتاج برمجة الـ Backend أصلاً؟

📄 مقارنة المواقع الساكنة والديناميكية

📄 Static Website (موقع ساكن)

  • بيعرض نفس المحتوى لكل الناس بلا أي تغيير.
  • الصفحات عبارة عن ملفات HTML ثابتة متخزنة على السيرفر.
  • مناسب للمواقع البسيطة التعرفية اللي مش بتتحدث كتير.
  • عشان تعدل كلمة، لازم تدخل تعدل ملف الـ HTML يدوياً وترفعه تاني.
<h1>Welcome to the department</h1>
<p>Office hours: 9:00 AM - 2:00 PM</p>

⚡ Dynamic Website (موقع ديناميكي)

  • المحتوى بيتغير بناءً على المستخدم أو البيانات المتسجلة.
  • الصفحة بيتم توليدها لحظياً بواسطة كود الـ Backend قبل ما تتبعت للمستخدم.
  • ضروري للمتاجر الإلكترونية، بوابات الطلاب، ولوحات التحكم.
  • البيانات بتيجي من الـ Database وتتعرض حسب الشخص اللي عامل Login.
def dashboard(request):
    student_name = 'Mona'
    cgpa = 3.8
    return render(request, 'dashboard.html',
        {'name': student_name, 'cgpa': cgpa})

Why Python & Django? — ليه اخترنا Python و Django؟

لغة قوية وإطار عمل متكامل لإنتاجية سريعة

🐍 مميزات لغة Python في الـ Backend
  • سهولة القراءة والفهم: الكود قريب جداً من اللغة الإنجليزية الطبيعية ومفهوم للمبتدئين والمحترفين.
  • الإجبار على تنظيم الكود: الـ Indentation (المسافات البادئة) جزء أساسي من الـ Syntax بتاع اللغة، وده بيخلي الكود نظيف ومنظم غصب عنك.
  • مكتبات ضخمة جداً: تدعم الويب، البيانات، الذكاء الاصطناعي، والأتمتة بكل سهولة.
  • مستخدمة في كبرى الشركات العالمية زي Google و Instagram و Spotify.
🏗️ What is a Framework? — يعني إيه Framework؟

الـ Framework هو عبارة عن مجموعة أدوات وقواعد وهيكل جاهز بيوفر على المطورين إعادة اختراع العجلة. بدل ما تبني كل حاجة من الصفر (زي التعامل مع الـ Database والـ Routing والحماية)، الـ Framework بيوفرهالك جاهزة.

💡 تشبيه بسيط: البناء بدون Framework = إنك تبني بيت وتبدأ تصنع الطوب والأسلاك والمواسير بنفسك. استخدام الـ Framework (زي Django) = إنك تشتري شقة كاملة المرافق فيها الحيطان والسباكة والكهرباء جاهزة، وتتفرغ أنت لفرشها وديكورها!
🛠️ إطار العمل Django ومفهوم الـ MVT

Django هو إطار عمل ويب عالي المستوى (High-level Python Web Framework) مبني بلغة Python بيشجع على التطوير السريع والأمان الفعال.

  • Batteries Included (بطاريات مدمجة): بييجي معاه أدوات جاهزة كتير زي نظام الصلاحيات، لوحة الإدارة (Admin Panel)، والـ ORM للتعامل مع الـ Database.
  • MVT Architecture: بيعتمد على نمط Model-View-Template:
    • Model: بيمثل هيكل البيانات والتعامل مع قاعدة البيانات.
    • View: بيمثل الـ Logic اللي بيعالج الطلبات ويقرر إيه الداتا اللي هترجع.
    • Template: بيمثل واجهة العرض (HTML) اللي بتتعرض للمستخدم.
🚨 نقطة امتحان مكررة: Django يعتمد على نمط MVT وليس MVC بشكل صريح (الـ View في Django بيقوم بدور الـ Controller في الـ MVC التقليدي، والـ Template بيقوم بدور الـ View).

Python Essentials — المتغيرات وأنواع البيانات في Python

حجر الأساس لكتابة أي كود برمجي بلغة Python

📦 المتغيرات (Variables) وقواعد التسمية

المتغير هو مكان محجوز في الذاكرة (Memory) لتخزين قيمة معينة واستخدامها لاحقاً في الكود.

🚨 قواعد تسمية المتغيرات في Python (مهمة للامتحان):
  • اسم المتغير لا يمكن أن يبدأ برقم (مثال: 1name خطأ ❌، لكن name1 صحيحة ✅).
  • اسم المتغير لا يمكن أن يحتوي على مسافات (مثال: student name خطأ ❌، بنستخدم الأندر سكور student_name بدالها ✅).
  • لا يمكن استخدام علامة الناقص أو الداش في التسمية (مثال: my-var خطأ ❌، الصح my_var ✅).
  • الاسم حساس لحالة الأحرف (Case-Sensitive) فـ Age غير age.
📊 أنواع البيانات الأساسية (Basic Data Types)
النوع الرمز مثال في الكود
نص (Text) str name = 'Ali' (بتتكتب بين علامات تنصيص single أو double quotes)
رقم صحيح (Integer) int level = 2
رقم عشري (Float) float gpa = 3.8
قيمة منطقية (Boolean) bool is_active = True (لازم الحرف الأول كابيتال: True أو False)
💬 التعليقات (Comments) في Python

التعليقات بنستخدمها عشان نوضح كود معين أو نكتب ملاحظات للمبرمجين، والـ Interpreter بتاع Python بيتجاهلها تماماً أثناء التشغيل.

  • التعليق المكون من سطر واحد بيبدأ بعلامة الهاش #.
  • مفيش في Python تعليقات بـ // زي JavaScript.
# ده تعليق سطر واحد ومش هيتنفذ
print("Hello, Backend World!")

User Input & Type Casting — مدخلات المستخدم وتحويل الأنواع

كيف نجعل البرنامج تفاعلياً ويتعامل مع الأرقام بحرية

⌨️ مدخلات المستخدم (User Input)

بنستخدم دالة input() عشان نخلي البرنامج يقف ويستنى المستخدم يكتب قيمة من الكيبورد.

⚠️ الفخ الشهير في الامتحانات: دالة input() بترجع دايماً قيمة نصية (String) بشكل افتراضي! حتى لو المستخدم كتب رقم 20، البرنامج هيشوفه كـ نص '20' ومش هيقدر يعمل عليه عمليات حسابية مباشرة.
🔄 تحويل الأنواع (Type Casting)

عشان نحل مشكلة إن الـ Input بيكون String دايماً، بنعمله تحويل نوع (Casting) للنوع اللي احنا عايزينه:

  • تحويل لنص: str()
  • تحويل لرقم صحيح: int()
  • تحويل لرقم عشري: float()
Casting Example
# قراءة العمر كـ String وتحويله فوراً لـ Integer
age = int(input("Enter your age: "))

# دلوقتي نقدر نجمع عليه أرقام عادي
print(age + 1)

Control Flow & Loops — اتخاذ القرارات والتكرار

توجيه مسار البرنامج وتكرار العمليات

⚖️ شروط الـ if / elif / else والـ Indentation

في Python بنستخدم شروط if عشان ننفذ كود معين بس لو الشرط تحقق. وبنستخدم elif (اختصار لـ else if) للشروط الإضافية، و else لو مفيش أي شرط تحقق.

⚠️ أهم قاعدة في لغة Python: المسافات البادئة (Indentation) إجبارية! لو نسيت تسيب مسافة بادئة (4 مسافات أو Tab) جوة البلوك بتاع الـ if أو الـ loop، الكود هيطلع IndentationError ومش هيشتغل.
Python Conditions
score = 75
if score >= 85:
    print("Excellent")
elif score >= 50:
    print("Pass") # المسافة البادئة دي إجبارية!
else:
    print("Fail")
🔄 التكرار (Loops): for و while
  • for Loop: بنستخدمها لما نكون عارفين عدد المرات اللي هنكرر فيها الكود مسبقاً (زي المرور على عناصر قائمة أو استخدام range()).
  • while Loop: بتفضل تكرر الكود طول ما فيه شرط معين بيساوي True. لازم تتأكد إن الشرط ده هيتغير جوة اللوب عشان ميعملش Infinite Loop (تكرار لانهائي) ويوقف السيرفر.
for loop with range
# هيطبع من 0 لـ 4 (الـ 5 مش معانا)
for i in range(5):
    print(i)
while loop with condition
count = 0
while count < 3:
    print(count)
    count += 1 # لو نسينا السطر ده، اللوب مش هتخلص!
🧩 المعاملات المنطقية (Logical Operators)

بنستخدمهم لربط كذا شرط مع بعض:

  • and: لازم كل الشروط تكون صح عشان ترجع True.
  • or: بيكفي إن شرط واحد بس يكون صح عشان ترجع True.
  • not: بتعكس القيمة المنطقية (بتحول الـ True لـ False والعكس).

Lists & Functions — القوائم والدوال في Python

تنظيم البيانات وإعادة استخدام الكود

📋 القوائم (Lists)

الـ List هي عبارة عن هيكل بيانات بيحفظ قيم كتير ورا بعض في متغير واحد. بتتكتب بين أقواس مربعة [] وبتتفصل العناصر بكومة ,. العناصر بتبدأ بـ Index رقمه 0.

List Example
students = ['Ali', 'Mona', 'Sara']
print(students[0]) # هيطبع Ali

# دالة pop() بتحذف العنصر الأخير وترجعه بشكل افتراضي
last_one = students.pop() # هيحذف Sara
⚙️ الدوال (Functions) والفرق بين print و return

الدالة هي بلوك من الكود ليه اسم وبياخد مدخلات ويقوم بوظيفة محددة عشان نقدر نكرر استخدامه بسهولة. بنعرفها بكلمة def.

🚨 الفرق بين print() و return (سؤال جوهري للمبتدئين):
  • print(): بيعرض النتيجة على الشاشة بس عشان المستخدم يشوفها. الدالة مش بتبعث القيمة دي لأي مكان تاني.
  • return: بتبعت القيمة الناتجة للبرنامج الرئيسي اللي نادى الدالة، عشان يقدر يسيفها في متغير تاني ويستخدمها في عمليات تانية.
Function with Parameters & Return
def add_numbers(a, b):
    return a + b

result = add_numbers(5, 10)
print(result) # هيطبع 15

Q1 Final 2021 & 2024
In a 3-tier web application, which layer does Django belong to?
A
Presentation Layer (Frontend)
B
Data Layer (Database)
C
Application Layer (Backend)
D
None of the above
✅ الإجابة الصحيحة: C — Application Layer
الـ Django والـ Python بيشتغلوا في الـ Backend اللي هو بيمثل طبقة المعالجة والمنطق (Application Layer). الـ HTML/CSS/JS بتمثل الـ Presentation Layer، والـ Database بتمثل الـ Data Layer. الـ Django هو قلب الـ Backend ومعالجة البيانات، عشان كده بيمثل الـ Application Layer أو الـ Logic Layer.
Q2 محاكاة للامتحانات
What is the main difference between static and dynamic websites?
A
Static websites show the same content to everyone; Dynamic websites change content based on user/database
B
Static websites need a database; Dynamic websites do not
C
Dynamic websites use HTML only; Static websites use Python
D
