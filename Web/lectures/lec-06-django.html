Django Models

Python classes that map to database tables

🗄️ Defining a Model
from django.db import models

class Student(models.Model):
    name = models.CharField(max_length=100)
    email = models.EmailField(unique=True)
    gpa = models.FloatField(default=0.0)
    enrolled = models.BooleanField(default=True)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.name
Key: Each model class = one database table. Each attribute = one column. Django ORM handles the SQL automatically.

Migrations

Syncing Python models with the database

🔄 Migration Workflow
1
Define or change models in models.py
Add new fields, change existing ones, create new Model classes.
2
python manage.py makemigrations
Creates migration files that describe the changes. Stored in migrations/ folder.
3
python manage.py migrate
Applies the migrations to the actual database. Creates/alters the real tables.
Exam: Migrations track changes to the database schema, are created with makemigrations, and applied with migrate — All of the above.

ORM Queries

Database operations without writing SQL

💾 Common QuerySet Operations
# Get all records
students = Student.objects.all()

# Filter records
top = Student.objects.filter(gpa__gte=3.5)

# Get single record
s = Student.objects.get(id=1)

# Create new record
Student.objects.create(name="Ali", email="ali@cu.edu")

# Update a record
s = Student.objects.get(id=1)
s.gpa = 3.9
s.save()

# Delete a record
s.delete()

Django Forms

Handling user input with built-in validation

📋 Form in View
from django import forms

class LoginForm(forms.Form):
    email = forms.EmailField()
    password = forms.CharField(widget=forms.PasswordInput)

# In the view:
def login_view(request):
    if request.method == 'POST':
        form = LoginForm(request.POST)
        if form.is_valid():
            # process the form data
            pass
    else:
        form = LoginForm()
    return render(request, 'login.html', {'form': form})
Pattern: GET → show empty form. POST → validate and process. Always check request.method == 'POST' first.

Django Admin Panel

Built-in management interface

⚙️ Registering Models in Admin
# admin.py
from django.contrib import admin
from .models import Student

admin.site.register(Student)   # ← register the model

# Create admin user:
# python manage.py createsuperuser
Access at: http://127.0.0.1:8000/admin/ — provides full CRUD for registered models with no extra code.

Django Part II — Exam Quiz

Real questions from Final 2021 — IS231

Q1 Final 2021
To register a model in the Django admin, you use:
A
admin.site.register(Model_name)
B
admin.register(Model_name)
C
models.admin.register()
D
site.register(admin, Model_name)
✅ Correct: A — admin.site.register(Model_name)
In admin.py, import your model and call admin.site.register(). This makes the model accessible in the Django admin interface at /admin/.
Q2 Final 2021
What are Migrations in Django?
A
They track changes to the database schema
B
They are created when you run makemigrations
C
They apply changes to the database using migrate
D
All of the above
✅ Correct: D — All of the above
Migrations: track schema changes (A), created by makemigrations (B), applied to DB by migrate (C). All three are correct aspects of Django migrations.
Q3 Final 2021
In Django's MVT, which component stores and manages data in database tables?
A
View
B
Template
C
Model
D
URLs
✅ Correct: C — Model
The Model in Django's MVT handles all data and database operations. Django ORM translates Python model classes into SQL database tables.
Q4 Final 2021
In Django, which template tag is used for defining HTML content (not outputting a variable)?
A
{{ tag }}
B
{% tag %}
C
{# comment #}
D
[% tag %]
✅ Correct: B — {% tag %}
{% %} is for template tags (logic: for loops, if conditions, block tags). {{ }} is for outputting variable values. {# #} is for comments.
Q5 Final 2021
When should you use POST method in a Django form?
A
When creating or updating data that changes the database
B
When reading data without changes
C
When you want URL parameters visible
D
When displaying a list of records
✅ Correct: A — When creating/updating data
Use POST for any operation that changes data (create, update, delete). Use GET for safe read-only operations where parameters in the URL are acceptable.
