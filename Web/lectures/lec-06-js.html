Array Methods

map, filter, pop — functional programming in JS

🗺️ map() — Transform Every Element
// map() applies a function to every element → returns NEW array
function funcX(num) { return num * 3; }
const Y = [1, 3, 4];
Y.map(funcX);   // → [3, 9, 12] — NEW array, original unchanged

// With arrow function
Y.map(n => n * 3);   // → [3, 9, 12]

// map() takes a FUNCTION as parameter (not a value!)
Y.map(n => n < 10);   // → [true, true, true] — correct
Y.map(n < 10);         // ❌ ERROR — not a function!
Exam: Y.map(funcX) where funcX multiplies by 3 → returns a NEW array [3, 9, 12]. The original array Y is NOT modified.
🔍 filter() — Select Elements
// filter() keeps only elements where function returns true
const numbers = [1, 5, 12, 3, 15];
numbers.filter(n => n < 10);   // → [1, 5, 3]

// CORRECT syntax (with function)
myNumbers.filter((value) => value < 10);   // ✅
// WRONG syntax (not a function)
myNumbers.filter(value < 10);              // ❌
Exam Tip: Both map() and filter() take a function as parameter. Passing a condition directly (not wrapped in a function) is incorrect.
🗑️ pop() — Remove Last Element
const arr = [1, 2, 3, 4];
arr.pop();     // removes and returns 4
console.log(arr);  // [1, 2, 3]
pop() always removes the last element from an array (opposite of shift() which removes the first).

Spread & Destructuring

Modern ES6+ patterns

... Spread vs Rest

Rest Parameter (...)

In function definition — collects multiple arguments into an array.

function f(...args) {
  // args is now an array
}

Spread Operator (...)

In function call — expands an array into individual arguments.

const arr = [1, 2, 3];
Math.max(...arr); // → 3
📦 Array Destructuring
const arr = [1, 2, 3, 4, 5];
const [x, ...k, y] = arr;
// x = arr[0] = 1 (index 0)
// y = arr[-1] = 5 (last index)
// k = [2, 3, 4] (everything in between)
Exam: The unpack method sets x to index 0, y to index -1 (last), and the rest to k.

CSS Selectors in JavaScript

Direct children, siblings, and relationships

🔍 CSS Child & Sibling Selectors
SelectorMeaningExample
A > BDirect children of Adiv > p selects <p> directly inside <div>
A ~ BAny sibling of Ah1 ~ p selects all <p> siblings after <h1>
A + BImmediately adjacent siblingFirst <p> after <h1>
Exam Tip: > = direct children. ~ = any sibling. These are CSS selector relationships.
🌲 DOM traversal — firstChild & nodeValue
// Access text inside an element
document.getElementById("id1").firstChild.nodeValue;
// firstChild = the text node inside the element
// nodeValue = the actual text string
Exam Note: Any element that accepts text always has a text node inside. firstChild accesses it, then nodeValue gets the contents.

AJAX & Fetch API

Asynchronous web requests without page reload

📡 What is AJAX?
  • AJAX = Asynchronous JavaScript and XML.
  • Allows web pages to update without a full page reload.
  • Modern AJAX uses JSON instead of XML, and Fetch API instead of XMLHttpRequest.
  • Common in: live search, chat apps, dynamic form validation.
🔗 Fetch API
// Basic fetch (GET request)
fetch('/api/students/')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

// With async/await (cleaner syntax)
async function getStudents() {
  const response = await fetch('/api/students/');
  const data = await response.json();
  console.log(data);
}

Q1 Final 2021
Consider function "funcX" that multiplies any input number by 3. You have Y = [1, 3, 4]. The result of Y.map(funcX) is:
A
New array [3, 9, 12]
B
Same array updated [3, 9, 12]
C
None of the answers
D
Something else
✅ Correct: A — New array [3, 9, 12]
map() always returns a NEW array. It does NOT modify the original array Y. The new array has each element multiplied by 3: [1×3, 3×3, 4×3] = [3, 9, 12].
For Q2-Q4: To select numbers less than 10 from myNumbers:
Q2 Final 2021
To select numbers less than 10 from array myNumbers, which is correct syntax?
A
myNumbers.map((value) => value < 10);
B
myNumbers.filter((value) => value < 10);
C
myNumbers.map(value < 10);
D
myNumbers.filter(value < 10);
✅ Correct: B — myNumbers.filter((value) => value < 10)
Use filter() to select elements. map() transforms elements (would return [true/false] not the actual numbers). The argument must be a function, not a raw expression.
Q3 Final 2021
In the function multiply(...myNumbers), the "..." are called:
A
Distribution parameter
B
Rest parameter
C
Arguments parameter
D
Waiting parameters
✅ Correct: B — Rest parameter
The ... syntax in a function parameter position is called a Rest Parameter. It collects all remaining arguments into an array.
Q4 Final 2021
CSS selector A > B selects:
A
Direct children of A
B
Any sibling of A
C
Any descendant of A
D
Adjacent sibling of A
✅ Correct: A — Direct children of A
A > B selects only direct children. A B (space) selects ALL descendants. A ~ B selects any sibling. A + B selects the immediately adjacent sibling.
Q5 Final 2021
CSS selector A ~ B selects:
A
Direct children of A
B
Any sibling of A
C
Any descendant of A
D
The parent of A
✅ Correct: B — Any sibling of A
A ~ B selects all sibling elements B that come after A in the DOM. It's the general sibling combinator.
Q6 Final 2021
Any element that accepts text always has a text node inside. To access its text, we use:
A
firstChild.nodeValue
B
innerHTML
C
textNode.value
D
firstChild.text
✅ Correct: A — firstChild.nodeValue
firstChild accesses the text node inside an element. nodeValue then accesses the actual text content of that node.
Q7 Final 2021
What does pop() do on a JavaScript array [1, 2, 3, 4]?
A
Removes the first element (1)
B
Removes the last element (4)
C
Removes a random element
D
Adds an element to the end
✅ Correct: B — Removes the last element
pop() removes and returns the LAST element. shift() removes the first. push() adds to the end. unshift() adds to the beginning.
Q8 Final 2021
The array [1, 2, 3, 4] after calling .pop() becomes:
A
[1, 2, 3]
B
[2, 3, 4]
C
[1, 2, 4]
D
[1, 2, 3, 4] (unchanged)
✅ Correct: A — [1, 2, 3]
pop() removes element at the end. The original array [1, 2, 3, 4] becomes [1, 2, 3] and pop() returns 4.
