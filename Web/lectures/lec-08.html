01

ليه محتاجين Database؟

Persistence: البيانات تعيش بعد ما البرنامج يقفل

Without Database

  • البيانات اللي في variables بتعيش طول ما السيرفر شغال فقط.
  • لو عملت restart، البيانات المؤقتة بتضيع.
  • صعب تشارك نفس البيانات بين مستخدمين كتير بشكل منظم.
memory.py
student_list = ["Ali", "Mona"]
# after server restart, this list is gone

With Database

  • البيانات بتتخزن بشكل دائم داخل Tables.
  • تقدر تعمل Search, Filter, Update, Delete بسهولة.
  • نفس السجلات متاحة لأجزاء مختلفة من النظام.
idnameleveldepartment
1Ali2CS
2Mona2AI
امتحانيًا: Database Storage = Persistent and structured. Program Memory = Fast but temporary.
02

What Is a Django Model?

Python class describes stored data

Model = Blueprint for Data

الـ Model في Django هو Python class بيورث من models.Model. كل attribute جوه الكلاس بيتحول إلى Column في Database table، وكل object من الكلاس بيمثل Row.

Django CodeDatabase Meaning
class Student(models.Model)Table اسمه غالبًا students_student
name = models.CharField(...)Column
Student(name="Ali")Object / Row data
__str__Readable display فقط، مش Column
students/models.py
from django.db import models

class Student(models.Model):
    name = models.CharField(max_length=100)
    age = models.IntegerField()
    department = models.CharField(max_length=50)

    def __str__(self):
        return self.name
ليه بنورث من models.Model؟ عشان Django يعرف إن الكلاس ده Database model ويضيف له ORM abilities زي objects, save(), delete().
Where Do Models Live?

غالبًا الموديلات بتتكتب داخل models.py في app معينة، لأن كل app بتمسك جزء منطقي من النظام.

project tree
myproject/
├── manage.py
├── myproject/
└── students/
    ├── admin.py
    ├── apps.py
    ├── models.py      # models live here
    ├── views.py
    └── migrations/

04

Migrations Workflow

من model code إلى database table

What Are Migrations?

الـ Migrations هي ملفات Python بتسجل تغييرات شكل الـ Database: إنشاء جدول، إضافة عمود، تعديل نوع Field... إلخ. اعتبرها history منظمة لتغييرات الـ schema.

1
Edit model code
تعدل models.py وتضيف/تغير fields.
2
Run makemigrations
Django يعمل migration file بناءً على التغييرات، لكنه لا يغير database لسه.
3
Run migrate
Django يطبق ملفات migration على قاعدة البيانات فعليًا.
terminal
python manage.py makemigrations
python manage.py migrate
python manage.py showmigrations
python manage.py sqlmigrate students 0001
فخ متكرر: makemigrations ينشئ ملفات migration فقط. migrate هو اللي يطبقها على الـ database. سؤال 2025 بيحاول يبدل المعنيين.
Common Migration Problems
  • عدلت model ونسيت تعمل migrate.
  • أضفت field إجباري على table فيها data قديمة من غير default.
  • عدلت migration قديم بعد ما اتطبق.
  • نسيت تضيف app في INSTALLED_APPS قبل شغل models.
05

Relationships Between Models

الجداول الحقيقية مرتبطة ببعض

ForeignKey

Many-to-One: طلاب كتير ينتموا لقسم واحد.

ForeignKey
class Department(models.Model):
    name = models.CharField(max_length=100)

class Student(models.Model):
    department = models.ForeignKey(
        Department,
        on_delete=models.CASCADE
    )
OneToOneField

One-to-One: طالب واحد له profile واحد فقط.

OneToOne
class StudentProfile(models.Model):
    student = models.OneToOneField(
        Student,
        on_delete=models.CASCADE
    )
    phone = models.CharField(max_length=20)
ManyToManyField

Many-to-Many: الطالب يأخذ كورسات كثيرة والكورس فيه طلاب كثير.

ManyToMany
class Course(models.Model):
    title = models.CharField(max_length=100)

class Student(models.Model):
    courses = models.ManyToManyField(Course)
اختيار العلاقة: Department to Student = ForeignKey. Student to IDCard = OneToOneField. Student to Course = ManyToManyField.
06

Django ORM and CRUD

Object Relational Mapper: تكلم مع database بـ Python

ORM = Python بدل SQL في البداية

الـ ORM يخليك تتعامل مع records كأنها Python objects. Django يحول أوامرك إلى SQL خلف الكواليس.

Django shell examples
# Create
Student.objects.create(name="Ali", level=2, department="CS")

# Read
Student.objects.all()
Student.objects.filter(level=2)
student = Student.objects.get(id=1)

# Update
student.level = 3
student.save()

# Delete
student.delete()
فخ: objects بحروف صغيرة. في 2025 ظهر سؤال Post.Objects.all()، وده غلط لأن Objects capital.
07

Django Admin

واجهة جاهزة لإدارة البيانات

What Is Admin?

Django Admin هي built-in management interface بتسمح للـ admins يعملوا Add, Edit, Delete للـ records من غير ما تبني dashboard من الصفر.

1
Register the model in admin.py
لو ما سجلتش model، مش هيظهر في admin interface.
2
Run migrations
لازم tables تكون موجودة في database.
3
Create superuser
تحتاج username/password للدخول على /admin/.
students/admin.py
from django.contrib import admin
from .models import Student

admin.site.register(Student)
terminal
python manage.py createsuperuser
امتحانيًا: التسجيل يكون في admin.py وليس settings.py. وإنشاء admin user بالأمر createsuperuser وليس createuser.
Customize Admin with ModelAdmin

لو عايز الـ admin list تبقى أوضح، استخدم list_display, search_fields, list_filter.

admin.py
from django.contrib import admin
from .models import Student

@admin.register(Student)
class StudentAdmin(admin.ModelAdmin):
    list_display = ("id", "name", "level", "department")
    search_fields = ("name", "department")
    list_filter = ("level", "department")

Mini Project Design

تصميم بسيط متوافق مع المحاضرة: Department + Student + Course.

models.py
class Department(models.Model):
    name = models.CharField(max_length=100)

class Course(models.Model):
    title = models.CharField(max_length=100)
    code = models.CharField(max_length=10)
    credit_hours = models.IntegerField()
    description = models.TextField(blank=True)

class Student(models.Model):
    name = models.CharField(max_length=100)
    level = models.IntegerField(default=1)
    department = models.ForeignKey(Department, on_delete=models.CASCADE)
    courses = models.ManyToManyField(Course, blank=True)

Q1 Final 2024
Django is based on which framework?
A
MVC
B
MVVM
C
MVT
D
None of the previous
الإجابة: C - MVT
Django architecture اسمها Model-View-Template. Model للبيانات، View للمنطق، Template للعرض.
Q2 Final 2024 / 2025
For a model to be visible in the admin interface it has to be registered in which file?
A
settings.py
B
admin.py
C
views.py
D
None of the previous
الإجابة: B - admin.py
استخدم admin.site.register(ModelName) أو @admin.register(ModelName). التسجيل في settings.py إجابة خاطئة ظهرت كفخ.
Q3 Final 2024
If you have a Django application named members, and a model named Member, what is a correct syntax to import the model?
A
from members.models import Member
B
import Member from members
C
import members.Member
D
None of the previous
الإجابة: A
الموديل موجود داخل ملف models.py في app اسمها members، لذلك الاستيراد الصحيح هو from members.models import Member.
Q4 Final 2024
When you have done changes in a model in Django, which command has to be executed in order to make the changes take effect in the database?
A
python manage.py runmigrations
B
python manage.py executemigrations
C
python manage.py makemigrations
D
None of the previous
الإجابة في ورقة 2024: C - makemigrations
السؤال يقصد أول command بعد تعديل model. عمليًا الـ workflow الصحيح: makemigrations لإنشاء ملف migration ثم migrate لتطبيقه على database.
