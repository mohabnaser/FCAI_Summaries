What is Django?

A high-level Python web framework for rapid development

🚀 Django Overview
  • Django is built using Python and designed for rapid web development.
  • It is a Web Framework — not a programming language, not just software.
  • Follows the MVT (Model–View–Template) architectural pattern.
  • "Batteries included" — routing, forms, admin, ORM, auth all built in.
Exam 2021 & 2024: Django is a type of Web Framework based on the MVT pattern.
💡 Why Use Django Instead of Writing From Scratch?

Without Framework

Write routing, templates, DB access, security, admin all from scratch. Slow and error-prone.

With Django

All that is already organized. Just write your business logic. Faster, safer, cleaner.

🏗️

MVT Architecture

Model · View · Template — the Django pattern

📐 The Three Components

📦 Model

Handles data and database tables. Defines the structure of data. ORM translates Python classes to SQL tables.

⚙️ View

Receives requests, runs logic, fetches data from Model, returns an HTTP response (often using a Template).

🎨 Template

HTML files that control what the user sees. Uses Django Template Language: {{ var }}, {% for %}, {% if %}.

MVT vs MVC: Django's MVT ≈ MVC. View in Django acts like the Controller. Template in Django acts like the View in MVC.

Setup Workflow

10 steps from nothing to a running Django server

📋 Full Setup Steps
1
Open folder in VS Code
File → Open Folder → your_project_folder
2
Open terminal
Terminal → New Terminal or Ctrl + `
3
Verify Python installed
python --version
4
Create virtual environment
python -m venv venv
5
Activate venv (Windows)
venv\Scripts\activate — look for (venv) in terminal
6
Install Django
pip install django
7
Create Django project
django-admin startproject myproject
8
Enter project folder
cd myproject — manage.py must be here
9
Run development server
python manage.py runserver
10
Open in browser
http://127.0.0.1:8000/ — default Django dev server port

Project Structure

What Django generates and what each file does

🗂️ Generated Files
myproject/
├── manage.py          ← main command entry point
└── myproject/
    ├── settings.py    ← all project settings (DB, apps, etc.)
    ├── urls.py        ← root URL configuration
    ├── asgi.py        ← async server interface
    └── wsgi.py        ← web server interface
Key: Always run python manage.py ... from the folder that contains manage.py.
📦 Django Project vs Django App

Django Project

The overall container. Can have many apps inside. Created with django-admin startproject.

Django App

Self-contained module with own models, views, templates, URLs. Created with python manage.py startapp. A Django app is BOTH an "extended package with base=Django" AND "a python package with own components".

Exam 2021: "What is a Django App?" → Both A and B — it's both an extended Python package AND has its own components.

Views & URL Configuration

Connecting URLs to Python logic

🔗 A Simple View
from django.http import HttpResponse

def home(request):           # Every view takes 'request'
    return HttpResponse("Hello, Django!")
🗺️ URL Configuration (urls.py)
from django.urls import path
from . import views

urlpatterns = [
    path('', views.home, name='home'),
    path('students/', views.student_list, name='students'),
]
🎨

Django Templates

Dynamic HTML with the Django Template Language

📝 Template Variables & Tags
<!-- {{ }} for variables — replaced with Python values -->
<h1>Hello, {{ name }}!</h1>

<!-- {% %} for tags — logic and control -->
{% for student in students %}
  <p>{{ student.name }}</p>
{% endfor %}

{% if user.is_authenticated %}
  <p>Welcome back!</p>
{% endif %}
Exam 2021 & 2024: {{ name }} = variable (replaced with Python value). {% tag %} = logic (for, if, block).
🔄 forloop Attributes
AttributeDescription
forloop.counterCurrent iteration (1-indexed)
forloop.counter0Current iteration (0-indexed)
forloop.firstTrue on first iteration
forloop.lastTrue on last iteration
forloop.revcounterIterations remaining (1-indexed)
Exam 2021: Valid attribute is forloop.counter — NOT forloop.counter(). No parentheses! It's a property not a method.
⚙️ Django Admin Registration
# admin.py
from django.contrib import admin
from .models import Student

admin.site.register(Student)  # ← makes Student visible in admin panel

# Create admin user:
# python manage.py createsuperuser
Exam 2021 & 2024: To register a model in the Django admin: admin.site.register(Model_name). NOT admin.register() or site.admin.register()!

Django Part I — Final 2021

Real questions with Egyptian slang 🇪🇬

Q1 Final 2021
Django is based on which pattern?
A
MVC
B
MVVM
C
MVT
D
None of the Above
✅ Correct: C — MVT (Model View Template)
Django uses the MVT pattern. Similar to MVC: Model=data, View=logic (like Controller), Template=HTML (like View in MVC). Django = MVT مش MVC. الاتنين متشابهين بس Django بيسمي الأجزاء بشكل مختلف. اللي اسمه "View" في Django بيعمل شغل الـ Controller في MVC.
Q2 Final 2021
What is a Django App?
A
An extended package with base package Django
B
A Python package with its own components
C
Both A and B
D
None of the above
✅ Correct: C — Both A and B
A Django app is both: an extended Python package using Django as the base, AND a self-contained Python package with its own models, views, templates, and URLs. لما السؤال يقول "Both" أو "All of the above" في Django، الغالب هي الإجابة الصح. الـ app هي package Python بتستخدم Django كـ base، وفي نفس الوقت package بكل مكوناتها.
Q3 Final 2021
What are Migrations in Django?
A
They track changes to the database schema
B
Created by running makemigrations command
C
Apply changes to the database using migrate command
D
All of the above
✅ Correct: D — All of the above
Migrations do all three: track schema changes, created by makemigrations, applied by migrate. الـ migrations بتعمل الثلاثة حاجات دول. لما بتعمل حاجة جديدة في الـ model → makemigrations بتعمل ملف بالتغييرات → migrate بتطبقها على الـ database الفعلي.
Q4 Final 2021
What does {{ name }} mean in Django Templates?
A
Will be output as-is "{{ name }}"
B
Displays the word "name"
C
Replaced with the Python variable's value
D
None of the above
✅ Correct: C — Replaced with Python variable value
{{ name }} is a template variable. Django replaces it with the value from the view's context dictionary. الأقواس المزدوجة {{ }} في الـ template = متغير Python. Django بيبدل {{ name }} بقيمة الـ variable اللي اتبعتت من الـ view في الـ context dictionary.
Q5 Final 2021
Which is a valid Django template forloop attribute?
A
forloop.reverse
B
forloop.previous
C
forloop.counter
D
forloop.lastitem
✅ Correct: C — forloop.counter
Valid: counter, counter0, revcounter, first, last. The exam may show forloop.counter() with parentheses — that's wrong! No parentheses. الـ valid attributes هي: counter, counter0, revcounter, first, last. خد بالك: مش forloop.counter() بأقواس — بدون أقواس. وبرضو مش forloop.reverse أو forloop.previous — دول مش موجودين.
Q6 Final 2021
Django is a type of:
A
Programming Language
B
Software / Application
C
Web Framework
D
Database
✅ Correct: C — Web Framework
Django is a Python web framework. Python is the language. Django is a framework built with Python. Django مش لغة برمجة — Python هي اللغة. Django هو framework (إطار عمل) مبني بـ Python. الفرق زي الفرق بين اللغة الإنجليزية (Python) وكتاب مكتوب بيها (Django).
Q7 Final 2021
To register a model in the Django admin, you use:
A
admin.site.register(Model_name)
B
admin.register(Model_name)
C
site.admin.register(Model_name)
D
models.admin.register(Model_name)
✅ Correct: A — admin.site.register(Model_name)
The exact syntax is admin.site.register(). Import the model then call this in admin.py. الـ syntax الصح: admin.site.register(اسم الـ model). احفظ ده كويس لأنه بيجي في 2021 و 2024. مش admin.register() — لازم يكون admin.site.register().

Django — Final 2024

New questions from the 2024 final 🇪🇬

Q1 Final 2024
Django architecture consists of ______:
A
Model and View only
B
Model, View, and Template
C
Template and View only
D
Model and Template only
✅ Correct: B — Model, View, Template
Django's MVT = Model (data) + View (logic) + Template (HTML). All three are needed. MVT = Model + View + Template. الثلاثة مع بعض. السؤال ده مجاش في 2021 بشكل مباشر بس المفهوم ده ظهر في 2024.
Q2 Final 2024
To register a model in the Django admin panel, you write:
A
admin.site.register(Model_name)
B
admin.register(Model_name)
C
django.admin.register()
D
Model.admin.register()
✅ Correct: A — admin.site.register(Model_name)
This question appeared in BOTH 2021 and 2024. The answer is always admin.site.register(). نفس السؤال جاي في الامتحانين 2021 و 2024. الإجابة دايماً admin.site.register(اسم الـ Model). حفظوه كويس!
Q3 Final 2024
In Django templates, {% %} is used for:
A
Displaying a Python variable value
B
Defining HTML content / logic tags
C
Adding a comment
D
None of the above
✅ Correct: B — HTML content / logic tags
{% %} = template tags (for, if, block, load). {{ }} = display variable. {# #} = comment. Option A from the exam said "a is wrong because we want to define the HTML content." الفرق المهم: {{ }} بيعرض قيمة متغير. {% %} بيعمل logic زي for وif. {# #} للـ comments. لو في السؤال شفت "define HTML content" → إجابته {% %} مش {{ }}.
