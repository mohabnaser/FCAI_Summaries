01

Two Cautionary Tales — قصتين بيبتدوا بيهم المنهج

ليه دكتور الرملي بيبدأ بقصتين كوارث؟ علشان يثبت إن الـ Software مش مجرد كود — Software bugs بتقتل وبتفقّر.

🚀 القصة الأولى — Ariane 5 (يونيو 1996)

Ariane 5 صاروخ فضاء أوروبي تابع لـ ESA (European Space Agency). بعد الإقلاع بـ 37 ثانية فقط، الصاروخ خرج عن مساره وانفجر فوق غويانا الفرنسية، وضاع حوالي $370 مليون.

السبب؟ سطر كود واحد. تحويل من 64-bit float لـ 16-bit signed integer بدون فحص للـ overflow. السرعة الحقيقية للصاروخ كانت أكبر من اللي يقدر يتحط في 16-bit int.

Ada · Ariane 5 (slide 10)
-- Horizontal velocity bias as measured by sensor
-- is converted to a 16 bit int WITHOUT checking
P_M_DERIVE(T_ALG.E_BH) :=
  UC_16S_EN_16NS (TDB.T_ENTIER_16S
    ((1.0/C_M_LSB_BH) *
    G_M_INFO_DERIVE(T_ALG.E_BH)));
Java equivalent (slide 11)
public class Overflow {
  public static void main(String[] args) {
    int x;
    double d = 5000000000.0;
    x = (int) d;
    System.out.println("d=" + d
      + " / x=" + x
      + " / x+1=" + (x+1));
  }
}
// Output:
// d=5.0E9 / x=2147483647 / x+1=-2147483648

▶️ اضغط الزرار وشوف الـ overflow بيحصل ازاي:

double d
5,000,000,000.0
→
cast to int
2,147,483,647
→
+ 1 → flip!
-2,147,483,648

⚠️ الـ moralعدم وجود error handling لـ overflow في تحويل الأنواع = صاروخ بـ 370 مليون دولار راح في 37 ثانية. Performance ≠ Safety.
🗺️ القصة التانية — Death GPS

أزواج كبار في السن تابعوا تعليمات GPS على طريق ضيق وخطر، وانتهى المطاف بسيارتهم في وادي/منطقة وعرة بدون أي وسيلة للنجاة. ناس ماتت بسبب الـ GPS بيوديهم في طرق غلط.

💡 الدرسالـ Software اللي بياخد قرارات نيابة عن الناس لازم يتعمل بحذر شديد. حتى لو الـ algorithm شغال صح في 99% من الحالات، الـ 1% ممكن يكلف أرواح.
🎯 اللي يجمع القصتين
  • Software failures = real-world cost — مش مجرد bug في الـ terminal، ده ممكن يكلف فلوس وأرواح.
  • Testing is critical — لو كان فيه اختبار كويس، الـ overflow كان اتلاقى قبل الإطلاق.
  • Software Engineering موجود علشان كده — منهج منظم علشان نقلل الكوارث دي.
02

Software Chronic Crisis — أزمة الـ Software

المشاكل المزمنة في صناعة البرمجيات: مشاريع بتفشل، مواعيد بتتأخر، تكاليف بتزيد، ووظائف ناقصة.

📖 تعريف Software Crisis

الـ Software Crisis بيتضمن (وده اللي بيتسأل عليه في الامتحان):

  • ✅ Software fails to serve its purpose, or may cause harm.
  • ✅ Project delivers reduced functional deliverables (المشروع بيتسلم وظايف ناقصة).
  • ✅ Project cost overruns and schedule delays (تكلفة أكبر ومواعيد متأخرة).
⚠️ Trap امتحان شائع"Project staff leave the project before completion" — مش جزء من تعريف الـ Software Crisis. الناس بيسيبوا الشركة في أي مهنة، ده مش حاجة خاصة بالـ software.
📜 Timeline of Famous Software Failures
15 January 1990
AT&T Long-Distance Network Crash
شركة AT&T فقدت 70 مليون مكالمة وخسرت حوالي $1 billion. السبب؟ missing break statement in C switch — الكود كان بيقع في الـ case التانية بدون ما يخرج. هنشوف الكود الأصلي تحت.
1991 (Gulf War)
Patriot MIM-104 Missile Failure
صاروخ Patriot الأمريكي miscalculated the trajectory بتاع صاروخ Scud العراقي الجاي عليه، فلم يضربه. مات 28 جندي أمريكي. السبب: cumulative rounding error في حساب الوقت.
June 1996
Ariane 5 Rocket Self-Destruct
Number conversion overflow (شفناه فوق). 64-bit float → 16-bit signed int بدون فحص. الصاروخ انفجر بعد 37 ثانية من الإقلاع.
1985–1987
Therac-25 Radiation Overdose
جهاز علاج إشعاعي طبي أعطى مرضى جرعات إشعاعية مميتة بسبب race condition في الـ software. مات على الأقل 3 مرضى. مثال كلاسيكي لخطورة الـ software في الـ safety-critical systems.
🐛 الكود الأصلي لكارثة AT&T 1990

المهم تشوف الكود ده وتعرف السطر اللي فيه البق — السؤال ده اتسأل في امتحان Winter 2021 Q1.04.

المشكلة: "The program should NOT have dropped out of the switch statement if 'ring write buffer is not empty'". يعني الـ break في المكان الغلط.

C-like pseudocode · AT&T 1990
1  while (ring receive buffer not empty and side buffer not empty) DO
2    Initialize pointer to first message in side buffer
3    get copy of buffer
4    switch (message)
5      case (incoming_message):
6        if (sending switch is out of service) DO
7          if (ring write buffer is empty) DO
8            send "in service" to status map
9          else
10            break   // ⚠️ THE BUG — drops out of switch instead of just out of if!
11          END IF
12        process incoming message
13      break
14    END SWITCH
15    do optional parameter work
🔴 السطر 10 هو السببbreak في C/C++ بتخرج من أقرب switch أو loop. هنا المبرمج كان قصده يخرج من الـ if بس الفعلياً خرج من الـ switch كله، فما عملش process incoming message وبدأ الـ system يتعطّل.
03

No Silver Bullet — مفيش حل سحري

Brooks (1986): مفيش technology واحدة هتحل كل مشاكل الـ software. لازم نتعامل مع طبيعة الـ software اللي بتميزه.

🐺 "Werewolves" — مين هم؟

في كتاب Brooks الشهير، الـ "werewolves" هم المشاريع اللي بتبان عادية في الأول، لكن بعدين بتتحول لمشاكل معقدة ومرعبة — زي الإنسان اللي بيتحول لذئب فجأة.

📌 Winter 2025 Q1.21اللي بيتسأل عليه: ايه معنى الـ werewolves في Brooks' metaphor؟ الإجابة: Projects that seem normal but become complex and problematic.
🔄 Software Changeability

الـ Software بطبعه بيتغير كتير — وده ليه مميزات وعيوب:

  • ✅ It has advantages and disadvantages.
  • ❌ NOT: "software is intangible and lacks a physical presence" (ده concept تاني).
  • ❌ NOT: "software must adapt to existing hardware interfaces" (ده compatibility مش changeability).
  • ✅ It often leads to frequent modifications and deviations from the initial design.
📌 Winter 2023 Q1.03الإجابة الصح: Only I & IV — التغيير ليه مميزات وعيوب، وبيؤدي لتعديلات متكررة وانحراف عن التصميم الأصلي.
📉 Software Failures Stem From... — أسباب الفشل

من امتحان Winter 2023 Q1.02 — ايه الأسباب الحقيقية لفشل الـ software؟

السببصح / غلطالسبب
Poor management decisions or lack of decisions✓ صحالمدير اللي مش بياخد قرارات أو بياخدها غلط = كارثة.
Poor software development practices✓ صحكود وحش، مفيش testing، مفيش version control = فشل مضمون.
Limited budget for the software project✗ غلطميزانية محدودة مش هي السبب الرئيسي — ممكن تعمل كود ممتاز بميزانية قليلة.
Overconfidence in the software performance✓ صح"كل حاجة شغالة على جهازي" → في الإنتاج بتفجر.
📌 الإجابة الصحالأسباب الحقيقية هي I, II & IV فقط — Poor management + Poor practices + Overconfidence.
04

What is Software Engineering?

تعريفات رسمية مهمة جداً للامتحانات النظرية + لمحة تاريخية.

💾 What is Software?

الـ Software مش بس computer programs — هو:

  • Programs (الكود نفسه)
  • Documentation (التوثيق)
  • Configuration data (الإعدادات اللي بتشغّله صح)
⚙️ What is Engineering?

تطبيق المعرفة العلمية والتقنية مع الموارد الفيزيائية لتصميم وإنتاج materials, structures, machines, systems تحقق هدف محدد تحت قيود معينة.

🏗️ Software Engineering

3 تعريفات لازم تحفظهم:

  • IEEE 610.12: "Application of a systematic, disciplined, quantifiable approach to the development, operation, and maintenance of software."
  • Sommerville: "Designing, building and maintaining large software systems."
  • Parnas: "Multi-person construction of multi-version software."
📅 When Did Software Engineering Start?
  • المصطلح اتقال شوية مرات في الـ 1950s و 1960s.
  • اتشهر في NATO Software Engineering Conference, 1968.
  • الـ NATO هو اللي خلى الـ field يبقى معترف بيه دولياً.
🌟 What Makes Great Software? — 4 viewpoints

كل واحد عنده تعريف مختلف لـ "الـ Software الممتاز":

👤 Customer

بيحل المشكلة بتكلفة معقولة (وقت + موارد).

🙋 User

سهل التعلم، استخدامه efficient، وبيخلّص الشغل.

👨‍💻 Developer

سهل التصميم والصيانة، وتم نشره بنجاح.

👔 Manager

بيبيع أكتر ويرضي العملاء، وتكلفته في التطوير قليلة.

05

Coder vs Hacker vs Programmer vs Software Engineer

4 ألقاب الناس بتخلط بينها — هنا الفرق الحقيقي.

🛠️ Coder

بيقدر يخلّيه يشتغل... بس مش هيبقى شكله جميل. الـ code بيكون بدون structure أو design.

🥷 Hacker

low-level expert، عنده فهم عميق لمنطقة معينة بس. شاطر لكن متخصص جداً.

💻 Programmer

بيكتب كود وبيفهم algorithms. غالباً بيشتغل لوحده وبيشتغل كويس.

🎯 Software Engineer

أفضل generalist — بيستخدم systems ولغات مختلفة ويخلّيهم يتكلموا مع بعض. True professionals: بيشتغل مع الناس وبيتواصل كويس.

💡 الفرق الجوهريالـ Programmer بيكتب كود. الـ Software Engineer بيبني نظام كامل — بيخطط، يصمم، يطور، يختبر، يوثّق، ويتواصل مع فريق وعملاء.

Exam Question Bank — Lecture 1

8 أسئلة حقيقية من امتحانات 2021 → 2025 · اختار إجابتك واضغط Check Answers

Question 1 Fall 2021–22 · Q1.03
In the case of AT&T long-distance phone network crash (15/1/1990) which caused the loss of 70 million phone calls and $1B revenue loss — what was the reason behind this failure?
A Bad UI design that confuses the operator
B A number conversion error
C Delay in software delivery by the contractor
D Missing break statement in C switch statement
✅ الإجابة الصح: D — Missing break in C switch الـ AT&T crash بسببه سطر break في مكان غلط داخل else، فالـ control بتدخل في case تاني بدون ما يكمّل معالجة الـ message الحالي.
ليه باقي الإجابات غلط: A — مفيش علاقة بالـ UI، ده crash في الـ network logic. · B — Number conversion error دي حكاية Ariane 5 مش AT&T. · C — الـ delivery كان تم بالفعل، المشكلة كانت في الكود اللي اتسلم.
Question 2 Fall 2021–22 · Q1.04
Which of these items is NOT included in the definition of software crisis?
A Software fails to serve the purpose or may cause harm
B Project delivers reduced functional deliverables
C Project cost overruns and schedule delays
D Project staff leave the project before completion
✅ الإجابة الصح: D — Project staff leave الـ Software Crisis ليه 3 مظاهر بس: الـ software بيفشل في غرضه، وظائف ناقصة، تكلفة ومواعيد بتزيد. سيبان الموظفين ده مشكلة HR في أي مهنة، مش حاجة خاصة بالـ software.
ليه باقي الإجابات صح (يعني جزء من التعريف): A — Software fails to serve purpose — ده العنصر الرئيسي. · B — Reduced functionality — مظهر شائع. · C — Cost & schedule overruns — أشهر مظاهر الأزمة.
Question 3 Winter 2021 · Q1.03
In the case of Patriot MIM-104, a software bug caused the system to:
A Not being able to see the attacking missile
B Not showing the trajectory of the attacking missile on the screen
C Miscalculate the trajectory of the attacking missile
D Fail to fire at the attacking missile
✅ الإجابة الصح: C — Miscalculate the trajectory الـ Patriot شاف الـ Scud واطلق عليه، بس بسبب cumulative rounding error في حساب الوقت، حسب مكان الـ Scud غلط فالـ صاروخ راح في الفراغ.
A/B — الـ radar شغّال، الـ Patriot شاف الصاروخ. · D — الـ Patriot فعلاً اطلق صاروخه، بس المسار كان غلط فلم يصب الهدف.
Question 4 Winter 2021 · Q1.04
Which line of code caused the failure of AT&T telephone network? (The program should NOT have dropped out of the switch statement if "ring write buffer is not empty")
A Line 2
B Line 13
C Line 6
D Line 12
E Line 10
✅ الإجابة الصح: E — Line 10 الـ break في السطر 10 بيخرج من الـ switch بالكامل مش بس من الـ if. اللي كان المفروض يحصل: استمرار في معالجة الـ incoming message بعد الـ else. بدلاً من كده، اتعمل break وخرج من كل حاجة.
A (line 2) — مجرد initialization. · B (line 13) — الـ break ده الطبيعي للخروج من الـ case، مش هو السبب. · C (line 6) — مجرد if condition. · D (line 12) — هو اللي المفروض يتنفّذ بس مش بيوصله بسبب الـ break الغلط في line 10.
Question 5 Winter 2023 · Q1.02
Software failures stem from... اختار الإجابة الصح:
I. Poor management decisions or lack of decisions
II. Poor software development practices
III. Limited budget for the software project
IV. Overconfidence in the software performance
A Only I & III
B Only II, III & IV
C Only I, II & III
D Only III & IV
E Only I, II & IV
✅ الإجابة الصح: E — Only I, II & IV Software failures سببها قرارات إدارية وحشة، development practices سيئة، وoverconfidence (ثقة زيادة في إن كل حاجة ماشية). الميزانية المحدودة (III) مش سبب رئيسي — ممكن تعمل كود ممتاز بميزانية محدودة لو الباقي مظبوط.
أي إجابة فيها III (Limited budget) هي غلط بشكل مباشر.
Question 6 Winter 2023 · Q1.03
Software changeability...
I. It has advantages and disadvantages
II. It means that software is intangible and lacks a physical presence
III. It means that software must adapt to existing hardware interfaces
IV. It often leads to frequent modifications and deviations from the initial design
A Only I & III
B Only I, II & III
C Only I & IV
D Only II & IV
E Only III & IV
✅ الإجابة الصح: C — Only I & IV Changeability ليها مميزات (سهولة التطوير) وعيوب (انحراف عن التصميم). وبتأدي لتعديلات متكررة.
II — "intangible" ده intangibility، خاصية تانية للـ software مش الـ changeability. · III — التكيف مع الـ hardware ده compatibility، مش changeability.
Question 7 Winter 2025 · Q1.21
In Brooks' "No Silver Bullet" metaphor, what do "werewolves" represent in software engineering?
A Bugs that are difficult to track
B Projects that seem normal but become complex and problematic
C Magical tools that promise to fix everything
D Software engineers who over-complicate systems
✅ الإجابة الصح: B — Projects that seem normal but become complex الـ metaphor من فيلم الـ werewolf: الإنسان يبان عادي لكن بيتحول لذئب مرعب. كده المشاريع: تبدأ بسيطة وبعدين تتعقد بشكل كارثي.
A — الـ bugs مش الـ werewolves في الـ metaphor. · C — الـ "silver bullet" نفسه هو الـ tool السحري — الـ werewolves هم الضحية مش العلاج. · D — الـ metaphor عن الـ projects مش الـ engineers.
Question 8 Winter 2025 · Q1.22
What is common between the cases of software failure in Ariane 5 and London Ambulance System (LAS)?
A The company developing the software had no prior experience in this type of software
B A number conversion to a smaller type caused overflow and hence system failure
C Testing was inadequate and insufficient
D The decision to release the system was done on political not technical reasons
✅ الإجابة الصح: C — Testing was inadequate and insufficient Ariane 5 ما اتعملش لها testing على السرعة الفعلية للصاروخ الجديد (الكود كان من Ariane 4 وما اتعادش testing له). LAS اتطلقت بدون load testing كافي. الـ قاسم المشترك = ضعف الـ testing.
A — الشركات كان عندها خبرة، المشكلة في التطبيق مش الخبرة. · B — ده سبب Ariane 5 لوحده مش LAS. · D — السياسة مش العامل المشترك هنا.

🚨 Common Mistakes — أخطاء بتتعمل كتير في الامتحان

1. خلط القصص ببعض AT&T = missing break · Ariane 5 = number overflow · Patriot = miscalculated trajectory · Therac-25 = race condition. الـ examiner بيخلطهم في الـ MCQ علشان يلخبطك.
2. "Project staff leave" = software crisis؟ لأ، مش جزء من التعريف. دي مشكلة HR. الـ crisis ليه 3 مظاهر بس: fail purpose / reduced functionality / cost & schedule overruns.
3. Limited budget = سبب رئيسي للفشل؟ لأ. الميزانية المحدودة مش من أسباب فشل الـ software في امتحان CS251. الأسباب الـ 3 هي: Poor mgmt, Poor practices, Overconfidence.
4. Changeability = Intangibility؟ لأ، خاصيتين مختلفتين. Changeability = بيتغير كتير. Intangibility = ملوش وجود فيزيائي. الاتنين خصائص للـ software بس مش نفس الحاجة.
5. السطر الغلط في كود AT&T؟ السطر 10 (break داخل else). مش السطر 13 (ده break صح). دائماً اقرا الـ logic كله مش بس تشوف break وتختار.
