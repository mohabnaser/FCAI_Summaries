Examples of Bad Designs

قبل ما نتعلم الصح، خلينا نشوف الغلط. الـ doctor بيدي 7 أمثلة من الواقع.

⚠️ الـ 7 أمثلة الكلاسيكية
1. Inconsistent Doors (Push vs Pull)
بابين شكلهم شبه بعض، واحد للـ push والتاني للـ pull، والـ user لازم يجرب علشان يعرف — وده بيعمل مشكلة في الـ consistency.
2. Stove Knobs (Poor Layout)
4 burners في مربع لكن الـ knobs في صف أفقي. مش واضح أي knob يتحكم في أي burner. Mapping problem.
3. CFC Sign (Unclear Labels)
"GODDARD SPACE FLIGHT CENTER — CFC — GOAL 457000". معاني CFC و 457000 غير موضحة. Not self-evident.
4. Elevator Buttons (Poor Organization)
أرقام مكتوبة في عمودين مش منطقي. اليسار: 2, 1, 8, 9. اليمين: 7, 6, 5, 4, 3. ترتيب فوضوي.
5. NY Times Homepage 2012 (Cluttered)
معلومات كتيرة جداً في صفحة واحدة. الـ user مش عارف يبدأ منين. Cognitive overload.
6. Door with "PULL" written on push-style handle
الـ handle شكلها بيقول "اضغط"، بس مكتوب عليها PULL بقلم. Contradicting affordances — التصميم بيتحدى نفسه.
7. "Invalid password format" (No Help)
الـ system بيقول الـ password غلط بس ما بيقولش الـ format المطلوب. الـ user مش عارف يصلح ايه.
💡 الخلاصةBad UI = الـ user محتاج يفكر كتير، يحفظ تفاصيل، أو يجرب. Good UI = الـ system بيقول للـ user ايه يعمل وايه حصل بوضوح.
02

Usability — 3 Pillars

التعريف الـ ISO الرسمي + الـ 3 معايير (Effectiveness, Efficiency, Satisfaction).

📖 ISO Definition
Usability is the extent to which a system, product or service can be used by specified users to achieve specified goals with effectiveness, efficiency, and satisfaction in a specified context of use.
— ISO

"We can't predict a software system's usability without testing it with real users." — Ferre et al., 2001

🎯 Effectiveness

الـ degree اللي بيها الـ system بيحقق غرضه — هل الـ user يقدر يخلص المهمة بنجاح؟

مثال: Banking app — هل الـ user قدر يحوّل فلوس بدون مساعدة؟ نعم → effective.

⚡ Efficiency

كم بسرعة وكم بمجهود قليل الـ user يخلص المهمة؟

مثال: Banking app — تحويل فلوس في 2 clicks أحسن من 10 clicks. الـ 2 clicks أكتر efficient.

😊 Satisfaction

الـ degree اللي بيها الـ user مبسوط من التجربة. تقيس بالـ feedback أو surveys.

مثال: Intuitive, جميل بصرياً, easy to navigate → الـ user راضي.

📌 ملحوظةالـ 3 معايير دي مختلفة. ممكن app يكون effective (بيخلص الشغل) بس not efficient (بياخد وقت) و not satisfying (UI وحش). الـ goal: الـ 3 مع بعض.

Why Usability Matters

دور المطور · Common pitfalls · Business impact.

👤 User-Centered Design

تصميم software يتوافق مع احتياجات وتوقعات المستخدمين.

💼 Business Impact

Usability وحشة → users غير راضين → زيادة support costs → معدل تبني أقل.

🏆 Competitive Advantage

Usable software → user satisfaction أعلى → retention أعلى → نجاح في السوق.

👨‍💻 دور المطور في الـ Usability
  • Involvement in Design — Developers participate early in design phase to integrate usability from start.
  • Collaboration with Designers — Work with UX/UI designers لـ alignment بين technical & UX goals.
  • Iterative Development — تحسينات in cycles + continuous user feedback & testing.

04

System Usability Scale (SUS) ⭐

أهم metric في الـ usability. Winter 2025 Q1.16 سأل عن حساب SUS — هتقدر تحسبها بإذن الله.

📊 What is SUS?
  • Standardized questionnaire لقياس الـ usability.
  • Developed by John Brooke in 1986.
  • الأكثر استخداماً في الـ UX research.
  • 10 questions، تقييم من 1 (Strongly Disagree) لـ 5 (Strongly Agree).
📝 الـ 10 أسئلة (Odd = positive, Even = negative)
#QuestionType
1I think I would like to use this system frequently+
2I found the system unnecessarily complex−
3I thought the system was easy to use+
4I think I would need the support of a technical person−
5I found the various functions were well integrated+
6I thought there was too much inconsistency−
7I would imagine that most people would learn this system very quickly+
8I found the system very cumbersome to use−
9I felt very confident using the system+
10I needed to learn a lot of things before I could get going−
🧮 Scoring Formula ⭐
  1. For odd questions (1, 3, 5, 7, 9): subtract 1 from user response.
  2. For even questions (2, 4, 6, 8, 10): subtract user response from 5.
  3. This scales all values from 0 to 4.
  4. Sum all 10 converted values (range 0 to 40).
  5. Multiply by 2.5 → final score 0 to 100.
SUS Formula
// For each odd question (Q1, Q3, Q5, Q7, Q9):
converted = response - 1

// For each even question (Q2, Q4, Q6, Q8, Q10):
converted = 5 - response

// Final score:
SUS = (sum of all converted values) × 2.5

✅ SUS Pros
  • Quick & easy — دقايق قليلة.

05

UMUX — Usability Metric for User Experience

الـ "shorter SUS" — 4 أسئلة بس بدل 10. مفيد للـ rapid prototyping.

⚡ الـ 4 أسئلة (scale 1-7)
  1. [This system's] capabilities meet my requirements.
  2. Using [this system] is a frustrating experience.
  3. [This system] is easy to use.
  4. I have to spend too much time correcting things with this system.
🏆 UMUX Advantages
  • أقصر من SUS (4 vs 10 أسئلة).
  • Strong correlation with SUS — لو SUS هتطلع كذا، UMUX هتطلع زيها تقريباً.
  • Easy & quick — مثالي للـ rapid prototyping أو early-stage testing.

Nielsen's 10 Usability Heuristics ⭐

Jakob Nielsen 1994. الـ 10 guidelines الأشهر لتقييم الـ UI. لازم تحفظهم بأمثلة.

H1
👁️ Visibility of System Status
النظام دايماً بيـ keep users informed عن اللي بيحصل، بـ feedback مناسب وفي وقت معقول. مثال: progress bar للـ download.
H2
🌍 Match Between System and Real World
استخدم لغة المستخدم — كلمات وأفكار مألوفة. مثال: iOS Compass app بيشبه الـ compass الحقيقي.
H3
🔄 User Control and Freedom
يقدر الـ user يـ undo / redo، يخرج، أو يلغي. مثال: confirmation قبل actions مهمة + recovery للملفات المحذوفة.
H4
📏 Consistency and Standards
نفس الـ terminology, layout, behavior في كل المنصة. مثال: 🔍 magnifying glass عالمياً = search.
H5
🛡️ Error Prevention
منع المشاكل قبل ما تحصل أفضل من error messages بعدها. مثال: Ryanair بيعطّل الـ destinations اللي مفيش فيها رحلات.
H6
👀 Recognition Rather Than Recall
قلل الـ memory load — اعمل الـ objects والـ options مرئية بدل ما الـ user يحفظ. مثال: dropdown menus بدل ما الـ user يحفظ commands.
H7
⚡ Flexibility and Efficiency of Use
دعم الـ novice والـ expert users — shortcuts, customizations, accelerators. مثال: Amazon filters للـ power users.
H8
🎨 Aesthetic and Minimalist Design
ما تحطش معلومات غير ضرورية. كل وحدة معلومة بتتنافس مع الباقي. مثال: Google Maps بيركز على الخريطة، الـ UI controls صغيرة.
H9
🚨 Help Recognize, Diagnose, Recover from Errors
Error messages تكون clear, helpful, تعطي حلول. لغة بسيطة. مثال: IKEA بيقول "Password needs 8 chars + uppercase + number".
H10
📚 Help and Documentation
حتى لو الـ system سهل، الـ help لازم تكون متاحة بسهولة. مثال: Airbnb Support section في الـ Profile tab.
📌 Winter 2025 Q1.17السؤال بيدي 4 icons (bell with badge, shopping cart, step indicator, battery level) ويسأل أيهم بيظهر system status. الإجابة: All four — كلهم بيعرضوا status (notifications, items in cart, current step, battery level).

Accessibility — Definition (ISO)

الفرق بين Usability و Accessibility: Usability = users محددين. Accessibility = أوسع نطاق ممكن.

📖 ISO Definition ⭐
Accessibility is the extent to which product, system, services, environments, and facilities can be used by people from the population with the widest range of characteristics and capabilities to achieve a specified goal in a specified context of use.
— ISO
📌 Winter 2025 Q1.18الـ definition دي بالظبط هي السؤال. الكلمة المفتاحية: "widest range of characteristics and capabilities". الإجابة: Accessibility.
🎯 Usability

Targeting specified users in a specified context.

مثال: app لمحاسبين الشركات — الـ usability بتقيس قد ايه المحاسبين بيستخدموه بسهولة.

🌍 Accessibility

Targeting widest range من الـ users — بمختلف القدرات والظروف.

مثال: نفس الـ app — هل يقدر أعمى يستخدمه بـ screen reader؟ هل يقدر من معاه إعاقة حركية يستخدمه بكيبورد بس؟

POUR — 4 Legal Requirements ⭐

الـ Accessibility Regulations 2018 (UK Public Sector). 4 شروط للموقع/التطبيق يبقى accessible.

P
Perceivable
المعلومات قابلة للإدراك بالحواس
O
Operable
الـ interface قابل للتشغيل
U
Understandable
المحتوى مفهوم وواضح
R
Robust
متوافق مع technologies المساعدة
👁️ P — Perceivable

المعلومات والـ UI components لازم تكون قابلة للإدراك بالحواس (بصر، سمع، لمس) بغض النظر عن قدرات المستخدم.

  • Text Alternatives — كل image تكون عندها alt tag للـ screen readers.
    <img src="search.png" alt="Search button">
  • Color Contrast — high contrast بين النص والخلفية لـ color-blind users.
  • Audio Content — captions أو transcripts للـ video/audio للـ deaf/hard-of-hearing.
⌨️ O — Operable

الـ interface لازم تكون قابلة للتشغيل بمختلف وسائل الإدخال (keyboard, mouse, touch, voice).

  • Keyboard Accessibility — كل function متاحة بالـ keyboard (Tab + Enter).
  • Touchscreen Support — buttons كبيرة كفاية و spaced للـ motor impairments.
  • Avoid Time Limits — أو provide خيار لتمديدهم.
🧠 U — Understandable

المحتوى والتشغيل لازم يكونوا واضحين وسهل فهمهم — حتى للـ cognitive disabilities أو الـ limited tech knowledge.

  • Easy Language — "Enter your email" بدل "Please provide your contact information".
  • Predictable Navigation — الـ menu في نفس المكان على كل صفحة.
  • Form Field Labels — كل field عنده label واضح.
💪 R — Robust

الـ system لازم يكون متوافق مع مختلف assistive technologies (screen readers, voice recognition).

  • يشتغل على desktop وmobile.
  • يدعم browsers مختلفة.
  • يشتغل مع JAWS, NVDA, VoiceOver.
09

Inclusive Computing

Design technology لـ أوسع نطاق ممكن من الناس. 15% of global population = 1 billion people.

🌐 6 أبعاد الـ Inclusion

🧠 Cognitive

Dyslexia, learning disabilities

👁️ Sensory

Vision, hearing impairments

♿ Physical

Motor impairments, wheelchair users

🗣️ Language

Non-native speakers

🌎 Cultural

Different backgrounds

💰 Socioeconomic

Income, education differences

💡 Why Does It Matter?
  • 15% of global population = some form of disability.
  • لو الـ tech مش inclusive = 1 billion people can't access our tools.
  • Inclusivity → better innovation. لما تصمّم للجميع، بتدفع حدود الـ tech.
Tim Berners-Lee (inventor of the Web): "Power of the web is in its universality. Access by everyone, regardless of disability, is an essential aspect."
🎤 Case Study #1: Google's Project Euphonia
  • الـ Speech recognition systems trained on "standard" speech.
  • الناس مع speech impairments ما تقدرش تستفيد.
  • Project Euphonia بيدرّب AI على non-standard speech.
  • بيحسّن قدرة الـ voice recognition للـ accessibility.
⚖️ Case Study #2: Domino's Pizza (2019)
  • Guillermo Robles — أعمى، ما قدرش يستخدم Domino's website/app بـ screen reader.
  • Domino's: حجتها إن الـ ADA (1990) ما بتغطيش الـ internet.
  • U.S. Supreme Court: رفض الحجة. الـ ADA applies to digital platforms.
  • الدرس: Digital accessibility = legal requirement not optional.
🏢 Microsoft Inclusive Design Toolkit — 3 Principles
  1. Recognize Exclusion — اعرف الـ bias والـ exclusions اللي بتحصل بسبب mismatch بين الناس والتجربة.
  2. Learn from Diversity — Inclusive Design بيحط الناس في المركز. perspectives متنوعة = insights حقيقية.
  3. Solve for One, Extend to Many — تصميم للـ permanent disabilities بينتج حلول تنفع الجميع.
💡 مثال "Solve for One, Extend to Many"Curb cuts (الـ dropped curb في الشوارع) اتعملت أصلاً للـ wheelchair users، لكن دلوقتي بتفيد: stroller users, cyclists, luggage rollers, everyone.

Exam Question Bank — Lecture 7

8 أسئلة من الـ امتحانات الحقيقية. SUS, Nielsen, Accessibility كلهم متكررين.

Question 1 Fall 2021–22 · Q1.15 (Message dialog)
A Message dialog box shows just "This is an error message" with an X icon and an OK button. What is wrong with this UI?
A Cluttered appearance and organization
B Unclear and uninformative error message
C Inconsistent colors
D Unclear what actions the user can take
✅ الإجابة الصح: B — Uninformative error message "This is an error message" ما بتقولش ايه الـ error ولا كيف تصلحه. ده violation لـ H9 (Help users recover from errors).
A — الـ dialog فيه element واحد، مش cluttered. · C — الألوان عادية. · D — في OK button، الـ action واضح.
Question 2 Fall 2021–22 / Winter 2021 · Q1.15 (Date dialog)
A Google Fit confirmation dialog shows a date picker with "Fri, Jun 15" and a "DONE" button, but no cancel option. Which UI design principle is violated?
A Use understandable encoding techniques
B Be consistent
C Provide all necessary help
D Ensure users always know what they can do next
E Ensure users can always get out, go back, cancel or undo actions
✅ الإجابة الصح: E — User can't cancel/go back ده violation مباشر لـ Nielsen H3 (User Control and Freedom). لو الـ user دخل بالغلط ومش عايز يحفظ تاريخ، مفيش طريقة يلغي.
A — الـ encoding (date) واضح. · B — الـ dialog consistent مع باقي Material Design. · C — مش عن مساعدة. · D — الـ user يعرف يضغط DONE.
Question 3 Winter 2025 · Q1.10
Which of the following best describes the principle of using understandable encoding techniques in UI design?
A Making sure the UI elements are visually appealing
B Organizing the UI elements in a hierarchical structure
C Using icons and symbols that are universally recognizable
D Implementing animations and transitions for a smoother user experience
✅ الإجابة الصح: C — Universal icons/symbols "Understandable encoding" = استخدام رموز عالمياً معروفة (🔍 search, ⚙️ settings, 🔔 notification). ده مرتبط بـ Nielsen H2 (Match Between System and Real World).
A — Visual appeal مهم بس مش "encoding". · B — Hierarchy عن الـ layout مش الـ symbols. · D — Animations عن الـ smoothness مش encoding.
Question 4 Winter 2025 · Q1.16 ⭐
A team conducted a SUS test and received these average responses (1=Strongly Disagree, 5=Strongly Agree):

Q1=4, Q2=2, Q3=5, Q4=2, Q5=4, Q6=2, Q7=5, Q8=2, Q9=5, Q10=1

What is the SUS score?
A 85 — Excellent usability
B 32 — Poor usability
C 68 — Average usability
D 34 — Poor usability
✅ الإجابة الصح: A — 85 (Excellent) Odd questions (Q1, Q3, Q5, Q7, Q9): response − 1
= (4−1) + (5−1) + (4−1) + (5−1) + (5−1) = 3+4+3+4+4 = 18

Even questions (Q2, Q4, Q6, Q8, Q10): 5 − response
= (5−2) + (5−2) + (5−2) + (5−2) + (5−1) = 3+3+3+3+4 = 16

Sum = 34. × 2.5 = 85.

الـ score > 80.3 → grade A → Excellent.
D (34) — ده هو الـ sum قبل × 2.5. لو نسيت تضرب في 2.5، هتختار D. · B, C — أرقام عشوائية.

Which of the following icons/UI elements show clearly the system status?
1. 🔔 Bell with red badge (notification count)
2. 🛒 Shopping cart with item count
3. Step progress indicator (Step 1 / Step 2 / Step 3 / Step 4)
4. 🔋 Battery level indicator (50%)
A Only 1, 2 and 4
B Only 1, 3 and 4
C Only 2 and 3
D All four

Question 7 Winter 2021 · Q1.16
Which UI design applies ALL these principles the most?

Question 8 Practice — Nielsen heuristic

A H1 — Visibility of System Status
B H4 — Consistency and Standards

🚨 Common Mistakes

1. نسيان ضرب SUS في 2.5الـ sum من 0-40. لو نسيت تضرب في 2.5، هتقدم 34 كإجابة بدل 85. دايماً اضرب في 2.5.
2. خلط الـ odd/even formulaOdd = positive items = subtract 1. Even = negative items = subtract from 5. اتذكر: "odd - 1, even from 5".
3. خلط Usability و AccessibilityUsability = specified users. Accessibility = widest range. الكلمات المفتاحية في الـ definition تختار الإجابة.
4. POUR — مش PORU أو POUEP-O-U-R: Perceivable, Operable, Understandable, Robust. "POUR" زي بصب water.
5. نسيان أمثلة الـ heuristicsالـ doctor بيدي صورة/scenario ويسأل أي heuristic. لازم تربط الـ heuristic بأمثلة (Cart icon = consistency, undo = freedom, etc.).

⭐ What Dr. El-Ramly Loves

🔥 الأسئلة المتكررة على Lec 7 (Winter 2025 جديد + قديم)
  1. SUS calculation — بيدي 10 responses ويطلب الـ score. احفظ الـ formula كويس.
  2. Nielsen heuristic violated — بيدي screenshot لـ bad UI ويسأل أي rule بيخالف.
  3. Encoding techniques — universal icons.
  4. Accessibility definition — الكلمة المفتاحية "widest range".
  5. POUR — كل principle مرتبط بمثال (alt text = perceivable, keyboard = operable).
  6. System status icons — أمثلة (battery, notifications, progress).
