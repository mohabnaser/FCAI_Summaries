Git's Distributed Model

Every developer has the FULL repository. Many operations are local.

🌐 Centralized vs Distributed

🏛️ Centralized (SVN, CVS, Perforce)

  • سيرفر واحد مركزي بيخزن كل النسخ.
  • الـ Clients بياخدوا نسخة يشتغلوا عليها (check out).
  • لازم نت (Network) عشان أغلب العمليات.
  • لو السيرفر وقع = الشغل كله بيقف.

🌳 Distributed (Git, Mercurial)

  • كل جهاز بيبقى عليه نسخة كاملة من الريبو.
  • أغلب العمليات local — مش محتاج نت عشان تشتغل.
  • السيرفر مجرد نقطة عشان نعمل sync ونجمع الشغل.
  • لو السيرفر وقع، الناس تقدر تكمل شغل عادي ونقدر نرجعه من أي نسخة.
🎯 2 Ways to Use Git
  1. لوحدك (Solo): على جهازك بس وعندك local repo وشغال مع نفسك.
  2. مع تيم (Team): أجهزة كتير + سيرفر (زي GitHub أو GitLab). وكل جهاز عليه نسخة كاملة من الريبو.
⚠️ NoteGitHub for personal projects is great, but for some academic work it might be restricted (e.g., to avoid making coursework public).
02

The 3 Local Areas ⭐

عندك التلاتة دول: Working Directory و Staging Area و Git Directory. هما دول قلب وعصب الـ Git.

📍 The 3 Areas
"Working Tree"

Working Directory

ده المكان اللي بتعدل فيه الفايلات. الفايلات هنا ممكن تبقى untracked، أو modified، أو unmodified.

"Index"

Staging Area

ده المكان اللي الفايلات بتتجهز فيه للـ commit الجاي. تعتبر لقطة (Snapshot) مستنية الحفظ.

"Repository"

Git Directory

المكان اللي الـ Git بيخزن فيه الهيستوري والكوميتس (اللي هو فولدر .git).

Operations between areas

From → To Command
Git Directory → Working Directory git checkout the project
Working Directory → Staging Area git add file
Staging Area → Git Directory git commit
03

File Status Lifecycle ⭐

Every file Git knows about is in one of 4 states.

🔄 The 4 States
untracked
add the file
unmodified
edit the file
modified
stage the file
staged
← commit sends staged files back to unmodified
← remove the file sends unmodified files back to untracked
❓ Untracked

الـ Git ميعرفش حاجة عن الفايل ده. فايل جديد أو منسوخ ومتعملوش add خالص.

عشان تخليه يتتبع: git add filename

✅ Unmodified

الـ Git عارفه، ومتغيرش من آخر commit. يعني نظيف ومفيهوش تعديلات جديدة.

✏️ Modified

الـ Git عارفه، بس إنت عدلت فيه بعد آخر commit، ولسه التعديلات دي مدخلتش الـ Staging Area.

📦 Staged

فايل إتعدل واتعمله git add. جاهز ومستني يدخل في الـ commit الجاي.

04

Local Git Commands

Complete cheat sheet for local repo operations.

⌨️ Setup & Status
$ mkdir site $ cd site $ git init Initialized empty Git repository in /home/user/site/.git/ $ ls -R .git branches/ config HEAD hooks/ info/ objects/ refs/ $ git status On branch master No commits yet nothing to commit (create/copy files and use "git add" to track)
📁 Staging Files
$ echo "hello" > myfile $ git status Untracked files: myfile $ git add myfile $ git status Changes to be committed: new file: myfile // To UN-stage a file: $ git rm --cached myfile
💾 Committing
// Add all + commit in one step: $ git commit -a -m "my commit message" // First-time identity: $ git config --global user.name "Your Name" $ git config --global user.email "you@example.com" // Amend (modify) the LAST commit: $ git commit --amend
💡 -a vs -am-a = stage all tracked changes (not new files). -m = message inline. Common: git commit -a -m "msg".
05

Branches ⭐

مسارات تطوير مستقلة. دي بقى الـ killer feature بتاعة الـ Git اللي مكسرة الدنيا.

🌿 Creating Branches
$ git branch b1 b2 * master
🌿 إنشاء الـ Branches
$ git branch b1 b2 * master $ git branch b3 # بتعمل برانش جديد $ git branch b1 b2 b3 * master

العلامة النجمة (*) دي معناها الـ branch اللي إنت واقف عليه حالياً.

🔀 التنقل بين الـ Branches
$ git checkout b1 Switched to branch 'b1' $ git branch -l * b1 master // Create + switch in one command: $ git checkout -b feature-branch Switched to a new branch 'feature-branch'
🔗 Merging Branches
// Switch to TARGET (the branch you merge INTO) $ git checkout master // Merge the source branch into it $ git merge feature-branch Updating 46cc7e3..3a4af86 Fast-forward file.txt | 7 +++++-- 1 file changed, 5 insertions(+), 2 deletions(-) // Delete branch after merge $ git branch -d feature-branch
💡 Fast-Forward MergeIf the target branch hasn't moved since the source branch was created, Git just "fast-forwards" the pointer. No new commit needed.
📂 Branch Strategies
  • Feature Branches — develop new features in isolation.
  • Release Branches — prepare a release.
  • Hotfix Branches — urgent production fixes.
06

Merge Conflicts

لما يتغير نفس السطر في أكتر من branch، هنا بيحصل الـ conflict وبتضرب مننا.

⚠️ Detecting a Conflict
$ git checkout b3 readme2.txt: needs merge error: you need to resolve your current index first

When Git can't auto-merge, it leaves conflict markers in the file like:

<<<<<<< HEAD
Conflicting change from current branch
=======
Conflicting change from incoming branch
>>>>>>> bugfix-branch
🛠️ Resolving Conflicts
  1. افتح (Open) الفايل اللي فيه الـ conflict.
  2. عدّله (Edit) بإيدك: اختار تسيب إيه وتمسح إيه، أو تدمجهم مع بعض.
  3. امسح (Remove) العلامات دي <<<, ===, >>>.
  4. اعمله Stage: git add filename
  5. اعمله Commit: git commit -m "resolved"
$ # after editing the file $ git add file.txt $ git commit -m "Resolve conflict and merge bugfix-branch"
07

Full Workflow Example

مسار كامل للـ Git من البداية للنهاية: init → commit → branch → merge → conflict → resolve.

🎬 Step-by-Step Sequence

1. Initialize

$ git init my-repo $ cd my-repo

2. Initial commit on master

$ echo "Initial Commit" > file.txt $ git add file.txt $ git commit -m "Initial commit" $ git log

3. Create and switch to feature branch

$ git checkout -b feature-branch $ echo "Feature Work" > feature.txt $ git add feature.txt $ git commit -m "Add feature work"

4. Merge feature back to master

$ git checkout master $ git merge feature-branch

5. Modify both branches → create conflict

$ echo "Hotfix in main" >> file.txt $ git commit -a -m "Apply hotfix" $ git checkout -b bugfix-branch $ echo "Bugfix Work" > file.txt $ git commit -a -m "Add bugfix work" $ git checkout master $ echo "Conflicting Change" >> file.txt $ git commit -a -m "Conflicting change in main" $ git merge bugfix-branch CONFLICT: Merge conflict in file.txt

6. Resolve conflict and finalize

# (edit file.txt manually, remove markers) $ git add file.txt $ git commit -m "Resolve conflict and merge bugfix-branch" $ git log --oneline --graph
08

git log — Viewing History

Many flags for different views.

📜 Common Flags
Flag الاستخدام (Purpose)
git log الافتراضي: بيعرض كل بيانات الـ commit، والجديد الأول
--pretty=short عرض مختصر (الهاش، المؤلف، والموضوع بس)
--pretty=full عرض كامل (المؤلف، الكوميتر، والرسالة كاملة)
--pretty=reference بيعرض بصيغة مرجعية (Reference format)
-3 آخر 3 كوميتس بس
--oneline سطر واحد لكل كوميت (الهاش + الرسالة)
--graph بيرسم شكل توضيحي للـ branches
--decorate بيضيف أسامي الـ branches والـ tags
--all بيعرض الكوميتس من كل الـ branches
🎨 Examples

git log --oneline

1234abc Commit message 1 5678def Commit message 2 9abc123 Commit message 3

git log --decorate

* 1234abc (HEAD -> master, origin/master) Commit message * 5678def (feature) Commit message

git log --graph --oneline --all

* 3a4af86 (HEAD -> master) Resolve conflict |\ | * 46cc7e3 (bugfix-branch) Add bugfix * | b7e2310 Hotfix in main |/ * 9abc123 Initial commit

أدوات زي Ungit بتديلك واجهة بصرية للموضوع ده — شجرة كوميتس تفاعلية (interactive commit tree).

09

Remote Repos — GitHub & GitLab

Shared projects, issue tracking, collaboration.

🐙 GitHub

أكبر منصة لاستضافة الكود. تبع شركة Microsoft.

  • ريبوز عامة وخاصة ببلاش.
  • بيوفر ميزات زي Issues و Pull Requests.
  • فيه GitHub Actions للـ (CI/CD).
  • فيه Marketplace و GitHub Pages.
🦊 GitLab

ممكن تسطبه عندك (Self-hostable) أو تستخدمه Cloud. بيوفر باكدج كاملة للـ DevOps.

  • الـ CI/CD pipelines متكاملة جواه.
  • فيه Issues و Merge Requests.
  • ممكن تسطبه على السيرفر بتاعك (Self-hosting).
  • فيه Container registry مبني جواه.
🧱 Common Remote Operations

Clone existing remote repo

$ git clone https://github.com/user/repo.git

Push to remote

# Standard workflow $ git branch new-branch $ git checkout new-branch # ...make changes, then: $ git add . $ git commit -m "Add new feature" $ git checkout master $ git merge new-branch $ git push

Pull latest changes

$ git pull // Fetches and merges remote changes into your current branch
🎫 Issues & Collaboration
  • Issues — عشان نتابع الـ bugs، نطلب features جديدة، ونتناقش.
  • Pull Requests (في GitHub) / Merge Requests (في GitLab) — تغييرات كود مقترحة ومعاها مراجعة (review).
  • Code Review — تعليقات، موافقات، وتغييرات مقترحة من التيم.
  • Labels & Milestones — عشان ننظم الشغل.

Practice Quiz — Lecture 18

8 أسئلة على Git mechanics, file lifecycle, branches, conflicts, remote.

Question 1 Practice — distributed
Which is the main difference between Centralized and Distributed VCS?
A Distributed is more secure
B Centralized supports branches; Distributed doesn't
C In Distributed, every machine has a full copy of the repository
D Distributed requires more disk space than Centralized (it doesn't)
✅ الإجابة الصح: C كل client للـ Git بيبقى عنده نسخة كاملة = فمعظم الأوامر بتشتغل local. الـ Centralized بيحتاج سيرفر عشان كل حاجة تقريباً.
Question 2 Practice — areas
In Git, what is the Staging Area?
A Where final commits are stored permanently
B A snapshot of changes prepared for the next commit (also called the "Index")
C Where you edit files
D The remote repository
✅ الإجابة الصح: B الـ Staging Area (أو الـ Index) دي اللي بتشيل الفايلات اللي جاهزة يحصلها commit. الـ git add بينقل الفايلات ليها.
Question 3 Practice — lifecycle
A file you just created but never added to Git is in which state?
A Modified
B Untracked
C Staged
D Unmodified
✅ الإجابة الصح: B — Untracked الـ Git ميعرفش حاجة عن الفايل لحد ما تعمله git add. بيبدأ Untracked → (بعد الـ add) → بيبقى Unmodified.
Question 4 Practice — branches
What does git checkout -b feature do?
A Lists all branches
B Deletes a branch
C Creates a new branch AND switches to it
D Renames a branch
✅ الإجابة الصح: C الـ -b معناها "اكريت branch جديد". والـ checkout بتنقلك ليه. الاتنين مع بعض = يكريت وينقلك عليه في خطوة واحدة.
Question 5 Practice — conflicts
After resolving a merge conflict, what is the correct next step?
A Revert the merge
B Run git merge again
C Push immediately without committing
D git add the resolved file, then git commit
✅ الإجابة الصح: D بعد ما تظبط الفايل اللي فيه الـ conflict وتمسح العلامات دي <<</===/>>>، بتعمله stage وبعدين commit. الـ commit ده هو اللي بيفنّش الـ merge.
Question 6 Practice — git log
Which flag shows commits from all branches in a visual graph with one-line summaries?
A --pretty=full
B --oneline
C --graph
D --graph --oneline --all
✅ الإجابة الصح: D اجمعهم مع بعض: --graph عشان تشوف الـ tree، و --oneline عشان تخليها مختصرة، و --all عشان تجيب كل الـ branches. دي أكتر تشكيلة بتنفعك.
Question 7 Practice — remote
To download an existing remote repository to your machine, you use:
A git init
B git clone <url>
C git pull
D git checkout <url>
✅ الإجابة الصح: B — git clone الـ clone بينزل نسخة كاملة من الـ remote repo لفولدر جديد عندك. الـ init بيكريت repo فاضي جديد. والـ pull بيسحب التحديثات في ريبو معموله clone أصلاً.
Question 8 Practice — fast-forward
A fast-forward merge happens when:
A The target branch hasn't moved since the source branch was created — Git just moves the pointer forward
B A merge results in a conflict
C You force push to a remote
D Two branches have diverged commits
✅ الإجابة الصح: A مفيش merge commit جديد بيتكريت — الـ pointer بتاع الـ target branch مجرد إنه بيمشي لقدام (fast-forwards) لحد آخر تحديث في الـ source branch. ده بينفع بس لو الـ target مفيش فيه أي commits جديدة.
